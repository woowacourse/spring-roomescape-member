# 1단계 - 테마 도메인 추가

## 기능 요구 사항

### 테마 관리

- [x] 관리자는 테마를 생성할 수 있다.
    - 테마는 `name`, `description`, `thumbnailUrl`을 가진다.
    - 클라이언트는 테마 생성 요청 시 `id`를 보내지 않는다.
    - 서버는 DB에서 생성된 `id`를 포함하여 생성된 테마 정보를 응답한다.

- [x] 관리자는 전체 테마 목록을 조회할 수 있다.
    - 등록된 모든 테마를 조회한다.
    - 각 테마는 `id`, `name`, `description`, `thumbnailUrl`을 포함한다.

- [x] 관리자는 테마를 삭제할 수 있다.
    - `themeId`를 기준으로 테마를 삭제한다.
    - 삭제 성공 시 응답 본문 없이 `204 No Content`를 반환한다.

### 예약과 테마 연결

- [x] 예약은 테마 정보를 포함한다.
    - 예약 생성 시 `themeId`를 함께 전달한다.
    - 예약 저장 시 `timeId`와 `themeId`를 각각 조회한 뒤 예약과 연결한다.
    - 존재하지 않는 `themeId`, `timeId`로 예약을 생성할 수 없다.

- [x] 예약 조회 응답에는 테마 정보가 포함된다.
    - 기존 예약 응답의 `time` 정보는 유지한다.
    - 추가로 `theme` 정보를 함께 응답한다.

# 2단계 - 사용자 예약

## 기능 요구 사항

### 예약 가능한 시간 조회

- [ ] 사용자는 테마와 날짜를 선택한다.
- [ ] 서버는 관리자가 등록한 전체 예약 시간 중 해당 날짜와 테마에 아직 예약이 없는 시간만 응답한다.
- [ ] 이미 예약된 날짜와 시간이더라도 테마가 다르면 예약 가능한 시간으로 응답한다.
- [ ] 예약 가능 여부는 클라이언트가 계산하지 않고 서버가 판단한다.

### 사용자 예약 생성

- [ ] 사용자는 예약 가능한 시간을 선택해 본인의 이름으로 예약할 수 있다.
- [ ] 예약 생성 요청은 `name`, `date`, `timeId`를 포함한다.
- [ ] 예약의 `themeId`는 URL 경로에서 전달한다.
- [ ] 예약 생성 응답은 생성된 예약의 `id`, `name`, `date`, `time`, `theme`를 포함한다.
