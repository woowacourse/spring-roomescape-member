# spring-roomescape-member

## 기능 요구 사항

지금까지는 정상 흐름(예약 가능 시간 조회 → 예약 생성)만 만들었다.
이번 사이클은 잘못된 요청에 대한 응답을 설계하고, 사용자가 본인의 예약을 조회·변경·취소할 수 있도록 발전시킨다.

이번 사이클의 작업도 백엔드 API 추가/변경과 사용자가 이해할 수 있는 화면 동작 두 가지를 함께 진행한다.
API와 에러 응답에 맞춰 사용자가 사용하는 클라이언트 화면을 직접 만들고, 잘못된 요청 시 사용자가 무엇이 잘못됐는지 알 수 있는 화면까지 확인한다.
화면 작성에는 AI를 활용해도 좋다.

### 1단계 - 서비스 정책 적용

다음 정책을 만족하지 않는 요청은 거부한다.

- 지나간 날짜·시간에 대한 예약 생성은 불가능하다.
- 같은 날짜+시간+테마에 이미 예약이 있으면 중복 예약을 거부한다.
- 예약이 존재하는 시간을 삭제할 수 없다.
- 유효하지 않은 입력값(빈 이름, 잘못된 날짜 형식 등)을 거부한다.

### 2단계 - 에러 응답 설계

- 서비스 정책 위반, 유효하지 않은 입력, 존재하지 않는 리소스 등에 대해 의도된 에러 응답을 반환한다.
- 500(서버 에러)이 사용자에게 노출되지 않도록 한다.
- 에러 응답 본문에 어떤 정보를 담을지 결정한다.
- 브라우저에서 에러 발생 시 사용자에게 의미 있는 메시지가 표시되어야 한다.

### 3단계 - 내 예약 조회/변경/취소

- 사용자가 자신의 이름으로 본인의 예약 목록을 조회할 수 있다.
- 사용자가 본인의 예약을 취소할 수 있다.
- 사용자가 본인의 예약의 날짜·시간을 변경할 수 있다.
- 변경·취소 시 발생하는 에러 케이스(이미 지난 예약을 취소, 변경하려는 시간이 이미 차 있음 등)도 2단계의 규칙에 맞춰 처리한다.

## 프로그래밍 요구 사항

- 자바 코드 컨벤션을 지키면서 프로그래밍한다.
    - 기본적으로 Java Style Guide을 원칙으로 한다.
- 레벨1에서 만든 규칙 중 좋은 코드 작성을 위해 의미있는 규칙을 정리하고 적용한다.
- 이번 사이클에서 구현한 API에 대한 요구사항 테스트를 작성한다.

## 기능 목록

- 사용자
    - 최근 1주 동안 예약이 많았던 인기 테마 10개를 조회한다. (예약 시점이 아닌 방탈출 날짜 기준)
    - 테마를 선택하면 예약 가능한 날짜가 표시된다.
    - 날짜를 선택하면 예약 가능한 시간이 표시된다.
    - 예약 가능한 시간을 선택하여 이름으로 예약한다.

- 관리자
    - 이름, 설명은 문자열로, 이미지는 첨부하여 테마를 추가할 수 있다.
    - 테마를 조회하여 삭제할 수 있다.
    - 예약 시간을 추가, 삭제할 수 있다.
    - 전체 예약 목록을 조회한다.

### 잘못된 입력

- 사용자
    - 과거의 예약을 생성하는 것
    - 중복 예약
    - 유효하지 않는 입력값
        - 빈 이름, 날짜 형식 불일치

- 관리자
    - 예약이 존재하는 예약 시간 삭제
