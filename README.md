# 사이클2 기능 목록

## 1. 에러 응답 규칙 정리
- 모든 예외 응답은 JSON 본문 형식으로 일관되게 반환한다.
- 사용자가 이해할 수 있는 메시지를 포함한다.
- 잘못된 요청에 500이 그대로 노출되지 않도록 처리한다.

## 2. 예약 생성 정책 적용
- 지나간 날짜/시간에 대한 예약 생성 요청을 거부한다.
- 같은 날짜, 시간, 테마에 이미 예약이 있으면 중복 예약을 거부한다.
- 빈 이름, 누락된 값, 잘못된 날짜 형식 등 유효하지 않은 입력값을 거부한다.

## 3. 예약 시간 정책 적용
- 예약이 존재하는 시간은 삭제할 수 없다.
- 예약 가능 시간 조회 시 지난 시간은 노출하지 않는다.

## 4. 내 예약 조회
- 사용자는 이름으로 자신의 예약 목록을 조회할 수 있다.
- 이름이 전달되지 않으면 전체 예약 목록을 조회할 수 있다.

## 5. 내 예약 취소
- 사용자는 자신의 이름과 예약 ID로 예약을 취소할 수 있다.
- 이미 지난 예약은 취소할 수 없다.
- 존재하지 않는 예약이거나 예약자 이름이 일치하지 않으면 취소할 수 없다.

## 6. 내 예약 변경
- 사용자는 자신의 이름과 예약 ID로 예약 날짜/시간을 변경할 수 있다.
- 이미 지난 예약은 변경할 수 없다.
- 지나간 날짜/시간으로는 변경할 수 없다.
- 변경하려는 시간에 같은 날짜, 시간, 테마의 예약이 이미 있으면 변경할 수 없다.
- 존재하지 않는 예약이거나 예약자 이름이 일치하지 않으면 변경할 수 없다.

## 7. 사용자 화면 반영
- 사용자 화면에서 이름으로 내 예약을 조회할 수 있다.
- 사용자 화면에서 내 예약을 취소할 수 있다.
- 사용자 화면에서 내 예약의 날짜/시간을 변경할 수 있다.
- 에러 발생 시 사용자가 이해할 수 있는 메시지를 화면에 표시한다.

# API 명세

## 예약 조회
- `GET /reservations`
- query parameter
   - `name` : (선택), 예약자 이름
- 동작
   - `name`이 있으면 해당 이름의 예약 목록을 조회한다.
   - `name`이 없으면 전체 예약 목록을 조회한다.

## 예약 생성
- `POST /reservations`
- request body

```json
{
  "name": "포비",
  "date": "2026-12-31",
  "timeId": 1,
  "themeId": 2
}
```

## 예약 취소
- `DELETE /reservations/{id}?name=포비`

## 예약 변경
- `PATCH /reservations/{id}`
- request body

```json
{
  "name": "포비",
  "date": "2026-12-31",
  "timeId": 3
}
```

# 에러 응답 명세

## 응답 본문 형식

```json
{
  "status": 409,
  "code": "RESERVATION_CONFLICT",
  "message": "선택한 시간은 이미 예약되어 있습니다.",
  "hint": "다른 시간을 선택해 주세요."
}
```

## 상태 코드 규칙
- `400 Bad Request`
   - 필수값 누락
   - 잘못된 날짜 형식
   - 잘못된 시간 형식
   - 유효하지 않은 요청 값
- `404 Not Found`
   - 존재하지 않는 예약
   - 존재하지 않는 테마
   - 존재하지 않는 예약 시간
   - 예약자 이름이 일치하지 않는 예약 변경/취소 요청
- `409 Conflict`
   - 지나간 날짜/시간 예약 생성 요청
   - 이미 지난 예약 변경/취소 요청
   - 중복 예약 생성/변경 요청
   - 예약이 존재하는 시간 삭제 요청
- `500 Internal Server Error`
   - 예상하지 못한 서버 오류
   - 사용자에게는 공통 안내 메시지를 반환한다.
