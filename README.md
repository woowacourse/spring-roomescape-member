## 1. 기능 요구 사항
### 1. 예약
1. 예약자와 관리자는 예약을 생성한다.
2. 모든 예약을 조회한다. 
3. 특정(본인의) 예약을 삭제한다.
4. 특정(본인의) 예약을 조회한다.
5. 특정(본인의) 예약을 변경한다.
6. 특정(본인의) 모든 예약을 조회한다.

### 2. 타임대
1. 타임대를 추가한다.
2. 전체 타임과 예약 가능 상태를 조회한다.
3. 특정 타임대를 삭제한다.

### 3. 테마
1. 테마를 생성한다.
2. 특정 테마를 삭제한다.
3. 테마들을 조회한다.(옵션: 날짜)
4. 1주 인기 테마들을 조회한다

### 4. 스케줄
1. 스케줄을 생성한다.
2. 특정 스케줄을 삭제한다.
3. 모든 스케줄을 조회한다.
4. 특정 스케줄을 삭제한다.

### 5. 예외 상황
1. 스케줄 테이블에 등록되지 않은 스케줄(날짜, 시간, 테마)로 예약을 시도할 시 예외가 발생한다.
2. 예약이나 스케줄에 등록되어있는 시간을 삭제하려할 시 예외가 발생한다.
3. 예약이나 스케줄에 등록되어있는 테마를 삭제하려할 시 예외가 발생한다.
4. 스케줄이 같은 예약을 중복예약할 수 없다.
5. 현재 날짜보다 지난 날짜의 스케줄로 예약을 할 수 없다.
6. 특정 값으로 조회시 데이터가 없다면 예외가 발생한다.
