# 사이클 1
## 기능 목록
### - 1단계 :

- [x] 예약에 테마 정보를 포함하도록 한다.
- [x] 테마는 이름, 설명, 썸네일 이미지 URL을 가진다.
- [x] 스케줄 시작 시간은 10시이고, 종료 시간은 20시이다.
- [x] 각 테마의 소요 시간은 2시간이다.
- [x] 관리자가 테마를 추가, 삭제할 수 있다.

### - 2단계 :

- [x] 사용자가 날짜와 테마를 선택하면 예약 가능한 시간 목록이 표시된다.
- [x] 사용자가 예약 가능한 시간을 선택하여 본인의 이름으로 예약한다. (동명이인은 어떡하지?)
- [x] 같은 날짜, 시간이라도 테마가 다르면 각각 예약 가능하다.

### - 3단계 :

- [x] 최근 1주 동안 예약이 많았던 테마 상위 10개를 조회한다.
  - 예: 오늘이 5월 8일이면, 게임 날짜가 5월 1일 ~ 5월 7일인 예약을 집계해 인기 순서대로 10개를 응답한다.

# 사이클 2
## 기능 목록 :
- [x] 지나간 날짜·시간에 대한 예약 생성은 불가능하다.
  - [x] 유효하지 않은 입력값을 거부한다.
- [x] 사용자가 자신의 이름으로 본인의 예약 목록을 조회할 수 있다.
  - [x] 유효하지 않은 입력값을 거부한다.
- [ ] 관리자는 예약이 존재하는 시간을 취소할 수 없다.
- [ ] 사용자가 본인의 예약을 취소할 수 있다.
  - [ ] 유효하지 않은 입력값을 거부한다.
  - [ ] 지나간 날짜·시간에 대한 예약 취소는 불가능하다.
- [ ] 사용자가 본인의 예약의 날짜·시간을 변경할 수 있다.
  - [ ] 유효하지 않은 입력값을 거부한다.
  - [ ] 같은 날짜+시간+테마에 이미 예약이 있으면 중복 예약이 안된다.
  - [ ] 예약이 존재하는 시간으로 변경할 수 없다.
  - [ ] 지나간 날짜·시간에 대한 예약 변경은 불가능하다.
