# 예약 변경/취소와 에러 처리

## 기능 구현 목록

### 1단계 - 서비스 정책 적용

- [x] 지나간 날짜에 대한 예약 생성 불가
- [x] 같은 날짜 + 시간 + 테마에 이미 예약이 있으면 중복 예약 거부
- [x] 예약이 존재하는 시간 삭제 불가 (DB 외래키 RESTRICT 설정)
- [x] 유효하지 않은 입력값 거부 (빈 이름, null 날짜/시간/테마)

### 2단계 - 에러 응답 설계

- [x] `@RestControllerAdvice`로 전역 예외 처리 추가
- [ ] 500 에러가 사용자에게 노출되지 않도록 처리
- [ ] 에러 응답 본문을 JSON 형식으로 통일
- [ ] 브라우저에서 에러 발생 시 사용자에게 의미 있는 메시지 표시

### 3단계 - 내 예약 조회/변경/취소

- [x] 이름으로 본인의 예약 목록 조회
- [x] 본인의 예약 취소
- [ ] 본인의 예약 날짜·시간 변경 API 추가
- [ ] 변경 화면 구현 (프론트엔드)
- [ ] 변경·취소 에러 케이스 처리 (이미 지난 예약, 변경하려는 시간이 이미 예약된 경우 등)

## API 명세

### 예약

| 메서드 | URL | 설명 | 성공 응답 |
|---|---|---|---|
| GET | /reservations | 전체 예약 목록 조회 | 200 |
| GET | /reservations/mine?name={name} | 이름으로 내 예약 조회 | 200 |
| POST | /reservations | 예약 생성 | 201 |
| DELETE | /reservations/{id} | 예약 취소 | 204 |

### 테마

| 메서드 | URL | 설명 | 성공 응답 |
|---|---|---|---|
| GET | /themes | 전체 테마 목록 조회 | 200 |
| GET | /themes/popular | 인기 테마 상위 10개 조회 (최근 1주일 기준) | 200 |
| POST | /admin/themes | 테마 생성 | 201 |
| DELETE | /admin/themes/{id} | 테마 삭제 | 204 |

### 시간

| 메서드 | URL | 설명 | 성공 응답 |
|---|---|---|---|
| GET | /times | 전체 예약 시간 목록 조회 | 200 |
| GET | /times/available?date={date}&themeId={themeId} | 예약 가능한 시간 조회 | 200 |
| POST | /admin/times | 예약 시간 생성 | 201 |
| DELETE | /admin/times/{id} | 예약 시간 삭제 | 204 |

## 에러 응답 설계

### 응답 형식

```json
{
  "status": 409,
  "message": "이미 예약된 시간입니다."
}
```

### 에러 케이스

| 상황 | 상태코드 |
|---|---|
| 중복 예약 | 409 Conflict |
| 지나간 날짜 예약 | 400 Bad Request |
| 존재하지 않는 리소스 | 404 Not Found |
| 예약이 존재하는 시간/테마 삭제 | 400 Bad Request |
| 유효하지 않은 입력값 | 400 Bad Request |
