# 방탈출 예약 관리 애플리케이션

## 1단계: 웹 요청-응답
### 구현할 기능
- [x] 예약 조회
- [x] 예약 추가
- [x] 예약 삭제

### 세부 요구사항
- [x] 별도의 데이터베이스 없이 메모리로 예약 상태를 관리한다.

## 2단계: 데이터베이스 연동
### 세부 요구사항
- [x] 기존 메모리 기반 관리를 데이터베이스 기반 방식으로 리팩터링한다.

## 3단계: 시간 관리
### 구현할 기능
- [x] 시간 추가
- [x] 시간 조회
- [x] 시간 삭제
- [x] 예약 추가 API의 요청 형태 변경
- [x] 예약 조회 API의 응답 형태 변경

#### 세부 요구사항
- [x] `reservation_time` 테이블로 시간 정보를 관리한다.
- [x] `reservation` 테이블의 `time` 컬럼을 `reservation_time` 테이블의 외래키로 변경한다.

## 4단계: 계층 분리
### 구현할 기능
- [x] 컨트롤러가 데이터베이스 계층을 직접 의존하지 않도록 리팩터링한다.


## 사이클 2 - 예약 변경/취소와 에러 처리
### 구현 기능 목록
1단계 - 서비스 정책 적용
- [x] 지나간 날짜와 시간에 대한 예약 생성 요청 시 예외처리
- [x] 같은 날짜 + 시간 + 테마에 이미 예약이 있으면 예외처리
- [x] 시간은 삭제할 수 있되, 예약이 존재하는 시간은 삭제 불가
- [ ] 유효하지 않은 입력값(빈 이름, 잘못된 날짜 형식 등) 입력 시 예외처리

2단계 - 에러 응답 설계
- [x] 서비스 정책 위반, 유효하지 않은 입력, 존재하지 않는 리소스 등에 의도된 에러 응답 반환
  - 500(서버 에러)이 사용자에게 노출하지 않는다
- [x] 에러 응답 본문에 담을 정보를 결정
  - 브라우저 에러 발생 시 사용자에게 의미 있는 메시지 표시

3단계 - 내 예약 조회/변경/취소
변경과 취소 시 발생하는 에러 케이스를 2단계 규칙에 맞춰 처리한다.
- [ ] 사용자가 자신의 이름으로 본인의 예약 목록을 조회하는 기능
- [ ] 사용자가 본인의 예약을 취소하는 기능
- [ ] 사용자가 본인의 예약의 날짜와 시간을 변경하는 기능


### 에러 응답 명세
Error Code: 400 Bad Request
```json
{
  "message" : "잘못된 문법으로 서버가 요청을 이해할 수 없습니다."
}
```

Error Code: 401 Unauthorized
```json
{
  "message" : "요청한 응답을 받기 위해서는 스스로 인증을 해야합니다."
}
```


Error Code: 403 Forbidden
```json
{
  "message" : "해당 콘텐츠에 접근할 권한이 없습니다."
}
```

Error Code: 404 Not Found
```json
{
  "message" : "요청받은 리소스를 찾을 수 없습니다."
}
```
