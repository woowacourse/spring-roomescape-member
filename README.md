# 사이클 2 요구사항

## 1단계 - 서비스 정책 적용

다음 정책을 만족하지 않는 요청은 거부한다.

- [x] 지나간 날짜·시간에 대한 예약 생성은 불가능하다.
- [x] 같은 날짜+시간+테마에 이미 예약이 있으면 중복 예약을 거부한다.
- [x] 예약이 존재하는 시간을 삭제할 수 없다.
- [x] 유효하지 않은 입력값(빈 이름, 잘못된 날짜 형식 등)을 거부한다.

## 2단계 - 에러 응답 설계

- [x] 서비스 정책 위반, 유효하지 않은 입력, 존재하지 않는 리소스 등에 대해 **의도된 에러 응답**을 반환한다.
  - **500(서버 에러)이 사용자에게 노출되지 않도록** 한다.
  - 에러 응답 본문에 **어떤 정보를 담을지 결정**한다.
  - 브라우저에서 에러 발생 시 사용자에게 **의미 있는 메시지가 표시**되어야 한다.
- 에러 응답은 [API 명세](docs/api)에서 확인할 수 있다.

## 3단계 - 내 예약 조회/변경/취소

- [x] 사용자가 **자신의 이름으로 본인의 예약 목록을 조회**할 수 있다.
- [ ] 사용자가 본인의 예약을 **취소**할 수 있다.
- [ ] 사용자가 본인의 예약의 **날짜·시간을 변경**할 수 있다.
- [ ] 변경·취소 시 발생하는 에러 케이스(이미 지난 예약을 취소, 변경하려는 시간이 이미 차 있음 등)도 2단계의 규칙에 맞춰 처리한다.

## 기능 요구 사항

### 테마 관리

- [x] 관리자는 테마를 생성할 수 있다.
    - 테마는 `name`, `description`, `thumbnailUrl`을 가진다.
    - 클라이언트는 테마 생성 요청 시 `id`를 보내지 않는다.
    - 서버는 DB에서 생성된 `id`를 포함하여 생성된 테마 정보를 응답한다.

- [x] 관리자는 전체 테마 목록을 조회할 수 있다.
    - 등록된 모든 테마를 조회한다.
    - 각 테마는 `id`, `name`, `description`, `thumbnailUrl`을 포함한다.

- [x] 관리자는 테마를 삭제할 수 있다.
    - `themeId`를 기준으로 테마를 삭제한다.
    - 삭제 성공 시 응답 본문 없이 `204 No Content`를 반환한다.

### 예약과 테마 연결

- [x] 예약은 테마 정보를 포함한다.
    - 예약 생성 시 `themeId`를 함께 전달한다.
    - 예약 저장 시 `timeId`와 `themeId`를 각각 조회한 뒤 예약과 연결한다.
    - 존재하지 않는 `themeId`, `timeId`로 예약을 생성할 수 없다.

- [x] 예약 조회 응답에는 테마 정보가 포함된다.
    - 기존 예약 응답의 `time` 정보는 유지한다.
    - 추가로 `theme` 정보를 함께 응답한다.

# 2단계 - 사용자 예약

## 기능 요구 사항

### 예약 가능한 시간 조회

- [x] 사용자는 사용자 이름으로 예약을 조회한다.
- [x] 사용자는 테마와 날짜를 선택한다.
- [x] 서버는 관리자가 등록한 전체 예약 시간 중 해당 날짜와 테마에 아직 예약이 없는 시간만 응답한다.
- [x] 이미 예약된 날짜와 시간이더라도 테마가 다르면 예약 가능한 시간으로 응답한다.
- [x] 예약 가능 여부는 클라이언트가 계산하지 않고 서버가 판단한다.

### 사용자 예약 생성

- [x] 사용자는 예약 가능한 시간을 선택해 본인의 이름으로 예약할 수 있다.
- [x] 예약 생성 요청은 `name`, `date`, `timeId`를 포함한다.
- [x] 예약의 `themeId`는 URL 경로에서 전달한다.
- [x] 예약 생성 응답은 생성된 예약의 `id`, `name`, `date`, `time`, `theme`를 포함한다.

# 3단계 - 인기 테마 조회

## 기능 요구 사항

### 인기 테마 조회

- [x] 사용자는 인기 테마 목록을 조회할 수 있다.
    - 인기 테마는 `GET /themes/popular`로 조회한다.
    - 각 테마는 `id`, `name`, `description`, `thumbnailUrl`을 포함한다.

- [x] 서버는 최근 일주일 동안 예약이 많은 테마를 인기 테마로 판단한다.
    - 예약의 생성일이 아니라 예약 날짜를 기준으로 집계한다.
    - 오늘 예약은 제외하고, 오늘 이전 날짜의 예약만 집계한다.
    - 최근 일주일 범위에 포함되지 않는 예약은 인기 테마 집계에서 제외한다.

- [x] 서버는 예약 수가 많은 순서대로 인기 테마를 응답한다.
    - 인기 테마는 최대 10개까지만 응답한다.

### 인기 테마 화면

- [x] 사용자는 홈 화면에서 인기 테마를 확인할 수 있다.
    - 홈 화면은 인기 테마를 캐러셀로 보여준다.
    - 집계된 인기 테마가 없으면 전체 테마 중 일부를 홈 캐러셀에 보여준다.

- [x] 사용자는 인기 방탈출 화면에서 인기 테마를 순위로 확인할 수 있다.
    - `/popular` 화면은 인기 테마 목록을 보여준다.
    - 집계된 인기 테마가 없으면 인기 테마가 없다는 메시지를 보여준다.
    - 사용자는 인기 테마에서 바로 예약 화면으로 이동할 수 있다.
