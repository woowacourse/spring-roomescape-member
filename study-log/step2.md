## 단계 2 페어 프로그래밍 회고

### 페어와 합의한 부분

1. 관리자 시나리오

1. 예약이 가능한 시간을 추가한다. 모든 테마가 추가된 시간에서 동시에 시작하고 동시에 끝난다.
    - POST /times

```
{
    start_at: "09:00"
}
```

2. 사용자 시나리오

1. 사용자가 원하는 날짜와, 원하는 테마를 선택한다.
    1. 날짜는 사용자가 원하는 날짜를 직접 입력하게 한다. (프론트에서 처리할 부분)
    2. 테마는 테마 리스트를 쭉 보여준다.
        - GET /themes
    3. 그중에서 테마를 고른다.
2. 선택한 날짜와 고른 테마에 대해 예약 가능한 시간 리스트를 받는다.
    - GET /times/{theme-id}?date=2026-05-05
3. 리스트 중 원하는 시간을 사용자가 선택하여 예약한다.
    - POST /reservations

### 미결 — 리뷰어 피드백을 받고 싶은 부분

#### 1. 화면을 만들어서 테스트할지, Postman으로만 진행할지

테스트 방식을 두고 고민이 있었다. 화면을 만드는 것은 AI를 사용할 수 있다고 해서 간단하게 처리할 수 있을 것 같다.

- 실무에서는 어떤 방식이 권장되는지 궁금하다.
- 우선은 백엔드 관점에 집중하기 위해 Postman으로만 테스트하기로 결정했다.
