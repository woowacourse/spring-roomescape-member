#### Request DTO 전달

클라이언트의 요청 구조가 서비스에 침투되지 않도록,  
변경 전파 최소화를 목표로 요청을 분해해 전달했으나  
`구구` 의 리뷰를 듣고 과연 그런가? 의문을 가지게 되었어요.

> 결국 Request 가 변경되면 컨트롤러에서 꺼내주는 코드도, 서비스가 받는 코드도 추가되네요?  
> 변경의 전파가 최소화된 것이 맞을까요?

변경의 전파, 계층간 분리라는 명목에 매인 오버엔지니어링인 걸까요?  
Request 를 서비스까지 넘겨주는 것이 최종 목적을 추구하는 실용적 예외인 걸까요?
