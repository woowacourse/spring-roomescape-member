### API 설계

`1단계`  
그룹 규칙은 아래와 같지만,
> 만약 사용자의 권한에 따라 응답 필드의 구성이나 노출 데이터가 현저히 다르다면  
> → 도메인 목적에 따라 API를 분리하거나 권한 로직으로 분기한다.

현재 구조에서 권한에 대한 분리가 존재하지 않기에  
1 단계에선 테마에 대한 접근이 관리자 전용임을 표시하기 위해  
/admin 을 붙이기보단 사용자의 테마 접근 가능성을 배제하고  
/theme 엔드포인트를 설정했어요.

`2/3단계`  
2/3 단계에선 별도의 엔드포인트를 설정하기보단  
사용자의 입장에서 조회하려는 `목적` 을 엔드포인트로,  
그에 필요한 `기준` 을 파라미터로 설계했어요.

API 설계 규칙/근거/결과물에 대한 리뷰와  
보완점/생각할 점에 대한 피드백을 받고 싶어요!
