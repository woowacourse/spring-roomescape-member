## 1️⃣ 학습법 v1

- 모르는 게 생기면 주변 크루에게 물어보며 이해가 될 때까지 질문하며 학습한다.
- 명확한 답이 나오지 않거나 도움 받을 크루가 없으면 기술 블로그, AI를 활용해 정보를 수집한다.
- 더 좋은 구현 방식이 있을 것 같은데 모르겠으면 우선 구현할 수 있는 방식으로 하고, 코드 리뷰를 통해 부족한 부분을 학습한다.
- 개념적인 이해가 있어도 코드 작성 방식을 모르겠을 때, 다른 크루의 코드를 보며 이해하는 방식으로 학습한다.

안 풀리는 지점: 당시에는 배운 것을 이해하는데, 학습 이후에 복습을 하지 않아서 금방 까먹는다.

## 2️⃣ 학습법 v2

1. 학습할 주제를 선택한다.
2. 수단을 가리지 않고 주제에 대한 정보를 습득한다.
3. 나만의 언어로 주제에 대한 설명을 간단하게 정리한다.

안 풀리는 지점: 나만의 언어를 검증하는 과정이 없어, 제대로 이해하지 못한 부분을 넘어갈 뻔했다.

## 3️⃣ 학습법 v3

1. 목표와 '학습의 깊이' 제한
2. 공식 문서 기반으로 정보 수집
3. 인지 과부하 시 '1-Depth 깊이 제한' 작동
4. '나만의 언어'와 '구체적 예시'로 정리하기
5. 마무리 검증: 개념 학습 전문가 AI 프롬프트를 활용해 오개념 테스트

- 프롬프트 예시

```
**"너는 학습자의 멘탈 모델(Mental Model)을 점검해주는 개념 학습 전문가야. 내가 어떤 기술을 공부하고 내 언어로 정리해서 가져올 거야. 너는 다음 두 가지에 집중해서 피드백을 줘.**

1. **오개념 체크:** 내 설명 중에 초보자가 흔히 오해하기 쉬운 지점이나, 개념적으로 틀린 부분이 있는지 바로잡아줘. 오개념은 없으면 억지로 생성하지 않아도 돼.

2. **경계 세우기:** 이 개념과 비슷해서 헷갈릴 수 있는 다른 개념과의 차이점을 명확히 할 수 있는 질문을 던져줘.


**너무 깊은 성능 최적화나 하드웨어 이야기는 빼고, 내가 이 개념의 '본질'과 '역할'을 정확히 이해했는지 확인하는 것에만 집중해줘."**
```

## ✏️ 주말에 시도할 것: "꼬리로 등장한 개념이 들어본 개념이면 궁금증 패스"

적용 시나리오:

1. 금요일 오후 ResponseEntity를 공식 문서를 토대로 학습한다.
2. 공식 문서에서 필요한 부분만 발췌한다.
3. 발췌한 문장에서 한 번이라도 들어본 개념이 나오면, ResponseEntity 이해에 필요한 정도만 보고 패스한다.
   (예시: 레이어드 아키텍처 공부 => 근데 아키텍처가 정확히 뭐지? ❌, 아키텍처는 한 문장으로 이런 거군~ ⭕️)
