# 학습법 v1

구현 중 모르는 개념이 나오면 크루들에게 질문하거나 인터넷 검색, AI 질문을 통해 즉시 해결책을 찾는다.
AI에게 내 수준에 맞는 학습 자료나 퀴즈를 요청하여 학습에 도움을 받는다.

미션 요구사항을 받으면 나름대로 객체의 역할과 책임을 고민하여 설계를 시도한다.
하지만 이론으로 공부한 내용을 실제 코드로 적용하는 과정에서 난관을 겪으면, 고민을 길게 이어가기보다 인터넷의 예시 코드나 AI가 제안하는 구조를 참고하여 기능을 완성하는 데
집중한다.

# 학습법 v2

1. 기술 구현 및 코드 학습
   AI한테 학습하고자 하는 기술의 수준별 문서를 달라고 하고, 기초 수준부터 이해하고, 일단 코드를 작성해본다. 이때, 코드를 복사 붙여넣기 하지 않는다.

코드를 작성한 후, 나의 코드의 근거와 문제점을 셀프 피드백 한다. 이후 나의 피드백을 AI에게 보내고, 내 피드백과 함께 코드 피드백을 받는다.
이때, 정답 코드를 받는 것이 아니라 내가 지금 시점에서 더 학습할 수 있는 질문 형식의 답변을 받는다.
질문을 받으면 나의 의견과 답변을 적어서 정리한다. 이 과정을 계속해서 반복한다.

다른 사람에게 개념을 설명할 수 있을 정도로 학습이 되면, 일단 학습을 종료한다.
다른 사람을 찾아가서 학습한 개념을 같이 토론해보거나, 혼자 말로 설명해본다. 하다가 막히는 부분이 있으면 그 부분을 기록하고 다시 위의 학습을 진행한다.

2. 개념 이해 및 구조화 학습
   AI에게 학습하고자 하는 개념의 수준별 문서를 달라고 하고, 그 문서에서 내가 원하는 결과물을 얻기 위해서는 어디까지 얻어야하는지, 어떤 부분을 얻어가야 하는 지를 물어본다.
   이때 학습의 시간을 정해서 물어본다. 왜냐하면 너무 방대한 양을 공부하려면 시간이 너무 많이 들고, 나중에 학습해도 좋을 부분에 시간을 너무 많이 할애해서 필요한 지식을
   제대로 학습하지 못할 수 있다.

AI에게 받은 문서 중 기초 수준의 문서부터 이해하고 이해한 내용을 백지에 적어본다.
내가 적은 내용을 AI에게 물어보고, 올바르게 이해하고 있는 지 피드백을 받는다.

다른 사람에게 개념을 설명할 수 있을 정도로 학습이 되면, 일단 학습을 종료한다.
다른 사람을 찾아가서 학습한 개념을 같이 토론해보거나, 혼자 말로 설명해본다. 하다가 막히는 부분이 있으면 그 부분을 기록하고 다시 위의 학습을 진행한다.

# 학습법 v3

1단계: 학습 단위 쪼개기

- AI 가이드 요청: 학습할 개념의 수준별 문서를 요청하되, "15~20분 내에 이해하고 백지에 요약할 수 있는 작은 단위로 목차를 쪼개달라”고 명시한다. 이때 수준별로 학습하며
  확실히 얻어가야 할 것에 대해서 설명받는다.
- 범위 및 시간 제한: "지금 당장 필요한 핵심은 무엇인가?"를 물어 학습 범위를 확정하고, 각 단위별로 20분 학습 / 5분 백지 복습 또는 코드작성 과 같은 타임라인을
  설정한다.

2단계: 작은 단위 학습

- 코드 학습 시:
    - 코드를 절대 따라치지 않고, 일단 코드를 나의 의견대로 타이핑한다.
    - 작은 기능 하나를 구현할 때마다 본인의 로직을 셀프 피드백한 후, AI에게 정답이 아닌, 더 잘 학습할 수 있는 질문을 요청한다.
- 개념 학습 시:
    - 쪼개진 학습 단위를 간단하게 학습하고 바로 백지에 구조화하여 적는다.
    - 작성한 내용을 소스 문서와 비교하여 AI에게 교정 받는다.

3단계: 지식의 연결 및 심화

- AI가 던진 질문에 자신의 의견을 달아 정리한다.
- 쪼개서 학습했던 작은 학습 단위들을 하나의 큰 그림으로 연결하여 정리한다.(마인드맵 등 활용)

4단계: 학습 종료

- 학습 종료 판정: "중학생에게 이 개념을 5분 안에 막힘없이 설명할 수 있는가?"를 기준으로 삼는다.
- 실전 토론 및 인출: 다를 크루들을 찾아가 학습한 내용을 토론하거나, 혼자서 소리 내어 설명해 본다.
    - 병목 구간 기록: 설명하다 막히는 부분은 학습 단위 설정이 잘못되었거나 이해가 부족한 지점이므로, 즉시 해당 부분만 다시 1단계로 돌아가 반복한다.

주말에 실제 시도할 1개: 일단 구현에 필요한 개념을 학습하기 위해서 AI에게 학습할 수 있는 코드를 요청한다.

3줄 적용 시나리오:

1. 토요일 오전 Spring Framework에서 HTTP 응답 학습 시, RequestEntity 를 사용하는 예제 코드를 AI에게 요청한다. 예제 코드를 따라 적으며 구조를
   이해하고, 사용법을 익힌다.
2. 다른 Spring Framework HTTP응답을 1번의 방법대로 학습한다.
3. 앞에 학습했던 개념이 정확히 기억나지 않고, 관성적으로 학습한다는 생각이 들면 멈춘다.
