### 학습 방법 v1
```text
1. 어느 정도 아는 범위에 대한 탐구
- 모르는 개념이 나오면 먼저 구글링을 통해 개념을 탐색했다.
- 하나의 개념을 깊게 이해해야 할 때는 공식 문서를 우선적으로 참고했다.
- 여러 개념을 비교해야 할 때는 블로그나 정리 글을 통해 빠르게 차이를 파악했다.

2. 정말 낯선 도메인을 접한 경우
- 학습 방향 자체가 잡히지 않을 때는 AI에게 전체적인 가이드라인을 요청했다.
- 이후 가이드라인을 기반으로 공식 문서 → 기술 문서 → 블로그 순으로 학습을 확장했다.

3. 정답이 없고 관점이 중요한 경우
- 주변 크루들과 대화를 통해 다양한 관점을 들으려고 했다.
- 내 생각과 다른 의견을 비교하면서 스스로 기준을 만들어보려고 했다.

또한 학습 과정에서 개념 자체보다  
“이 개념을 코드에 어떻게 적용해야 하는지”에서 자주 막혔다.

이럴 때는 AI에게 구체적인 코드 적용 방법을 물어보고,  
직접 내 코드에 적용해보면서 이해하려고 했다.
```

### 학습 방법 v2
```text
학습을 시작할 때 문제 인식 → 왜 필요한가? / 왜 공부하는가? 정의 → 공부할 내용 소분 → 적용 및 피드백 반복 → 구조 정리 → 모든 자원 활용해서 비교
```
### 학습 방법 v3
```text
1. 나만의 학습 방법 v3
문제 인식 → 실패 경험 → 최소 학습 → 적용 → 구조화 → 설명
-------------------------------------------------------
2. 구체적인 학습 방법
1단계: 문제 인식 (Problem-Driven)
Action: 실제로 내가 겪은 문제를 한 문장으로 정의한다.
Rule: 불편/버그/이상한 코드 경험 없으면 시작하지 않는다.

2단계: 실패 경험 (Fail First)
Action: 해당 원칙을 일부러 깨는 코드를 직접 작성한다. 또는 기존 코드를 일부러 더 망가뜨린다.
Goal: 왜 이걸 쓰는지 체감

3단계: 최소 학습
Action: 필요한 개념만 빠르게 학습한다. (문서 / 강의 / AI 활용)
Rule: 완벽 이해 금지, 지금 문제 해결에 필요한 만큼만 이해

4단계: 적용 & 리팩토링 (Core Loop)
Action: 2단계의 나쁜 코드를 직접 고친다. 개선 전/후를 비교한다.
AI 활용
- “이 코드가 왜 나아졌는지 설명해줘”
- “이 리팩토링의 한계는 뭐야?”

5단계: 구조화 (Structural Thinking)
Action: 아래 형태 중 하나로 정리
- Before / After
- 문제 → 해결 → 효과
- 개념 간 관계
Goal: 재사용 가능한 지식으로 만든다

6단계: 설명
Action: 한 문장 + 비유로 설명
예시
❌ “OCP는 확장에는 열려 있고 변경에는 닫혀있다”
✅ “기존 코드는 건드리지 않고, 새로운 부품만 끼우는 구조”
-------------------------------------------------------
[추가 - 여러 상황에서의 학습 방법]
1. 공통 루프
- 문제 인식
- 왜 필요한가
- 적용
- 피드백

2. 상황별로 적용
1) 개념/설계 학습 (SOLID, OOP)
- 안티패턴 → 리팩토링

2) 알고리즘
- 문제 유형 분류
- 풀이 패턴 축적
- 오답 노트

3) 프레임워크 / 기술 스택
- 실행 → 에러 → 해결 반복
- 흐름 추적 (요청 → 처리 → 응답)
- 작은 기능 단위 실습
```
