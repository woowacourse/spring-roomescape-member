# 학습법 변천사
## 학습법 v1
1. 구글링을 통해 모르는 키워드를 찾아본다.
2. 읽어보고 이해한 내용을 나만의 언어로 정리한다. (ex - 원시값 포장: 의미와 행동을 부여하고, 유효성을 검증할 수 있음)
3. AI에게 해당 내용이 맞는지 검증하고, 틀리면 수정한다.

## 학습법 v2
1. 구글링을 통해 모르는 키워드를 찾아본다.
2. 읽어보고 이해한 내용을 노션에 나만의 언어로 정리한다. 
   <br> a. 단순히 1줄로 정의하는 것이 아닌, 내가 실제로 해당 키워드가 필요했던 상황을 구체적으로 작성한다.
3. AI에게 해당 내용에 오류가 있는지 검증하고, 필요 시 수정한다.
4. 정리된 내용을 바탕으로, 해당 키워드에 대해 크루를 붙잡고 토론한다. (왜 필요할까? 어떤 상황에서 사용해야 할까? 이 방법 말고 다른 방법은 없을까? 각 방법의 장단점은 무엇일까?)
   <br> a. 아직 내가 잘 모르고 있다는 판단이 들면 2~3번을 반복한다.
5. 위 내용을 바탕으로 나만의 결론을 내린다.

## 학습법 v3
1. "이 키워드 없이 직접 실패를 만들 수 있는가?"를 먼저 판단한다.
   <br> a. Yes → 키워드 없이 직접 구현해보고, 막히거나 실패한 지점을 기록한다.
   <br> b. No → 2번으로 바로 넘어간다.
2. 구글링으로 키워드를 학습하고, 이해한 내용을 노션에 나만의 언어로 정리한다.
   <br> a. 단순 정의가 아닌, 내가 실제로 이 키워드가 필요했던 상황을 구체적으로 적는다.
   (1번을 거쳤다면, 실패했던 지점이 왜 실패였는지도 함께 적는다.)
   <br> b. AI에게 오류가 있는지 검증하고, 필요 시 수정한다.
3. 노션을 닫고, 빈 문서에 해당 키워드를 크루에게 설명한다고 생각하며 처음부터 써본다.
   <br> a. 막히는 부분에 물음표(?) 표시를 한다.
   <br> b. 물음표(?)가 생겼다면 2번으로 돌아가 반복한다.
4. 정리된 내용과 물음표(?) 를 바탕으로 토론한다. (왜 필요할까? 어떤 상황에서 사용해야 할까?)
   <br> a. 크루와 토론이 가능할 때 - 물음표(?) 를 의제로 크루와 토론한다.
   <br> b. 토론 상대가 없을 때 - AI에게 물음표(?) 를 들고 가되, "답을 바로 말하지 말고 힌트만 달라"고 명시한다.
5. 토론을 바탕으로 나만의 결론을 적는다.
   <br> a. 아직 답하지 못한 물음표(?) 를 다음 나선의 좌표로 남긴다. (레벨2, 혹은 다음 미션에서 이 키워드를 다시 만났을 때 파고들 출발점)
