### v4

```
ai 사용을 최소화하고 스스로 구현하고 이후 비교 학습한다.

학습은 목적 중심으로, 한 번에 하나씩 진행한다.
의문이 해소되면 깊게 파지 않고 다음 단계로 이동한다.

의문을 두 가지로 구분한다.
1. 바로 해결해야 하는 의문은 필요한 만큼만 이해하고 적용한다.
2. 구조를 이해해야 하는 의문 한 번은 끝까지 이해한다.

모든 의문을 해결하려 하지 않는다.
지금 목적과 직접 연결되지 않은 의문은 기록하고 넘긴다.
모든 의문을 파지 않고, "지금 필요한 의문 vs 나중에 다룰 의문"을 구분한다.
```

### v5

```
1. AI 사용은 최소화하고, 먼저 스스로 구현한다.
   구현이 끝난 뒤 블로그, AI, PR을 보고 다른 사람의 방법과 비교한다.
   서로의 장단점을 파악하고 흡수한다.

2. 학습은 목적 중심으로 진행한다.
   한 번에 하나씩만 학습하고 적용한다.
   지금 해결하려는 문제가 무엇인지 계속 리마인드한다.

3. 의문이 해소되면 불필요하게 깊게 파지 않는다.
   지금 목적을 달성하는 데 필요한 만큼만 이해하고 다음 단계로 이동한다.
   다만 구조를 이해해야 하는 문제는 한 번은 원리까지 확인한다.

4. 의문을 구분한다.
   - 지금 바로 해결해야 하는 의문
   - 구조를 이해해야 하는 의문
   - 지금 목적과 직접 연결되지 않은 의문

5. 모든 의문을 해결하려 하지 않는다.
   지금 목적과 직접 연결되지 않은 의문은 기록하고 넘긴다.
   이후 같은 주제를 다시 만났을 때 더 깊게 학습한다.

6. 여러 주제가 동시에 떠오르면 바로 해결하려 하지 않는다.
   먼저 문제를 분리한다. 
   분리한 문제 중 지금 목표에 직접 필요한 것부터 하나씩 해결한다.

7. 성격이 다른 문제를 섞어서 판단하지 않는다.
   예를 들어 테스트 통과를 위한 최소 정책과 실제 서비스 정책은 분리해서 생각한다.
   지금 해결할 문제와 나중에 다룰 문제를 구분한다.
```
