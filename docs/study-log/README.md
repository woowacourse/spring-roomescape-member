# 학습법 변천 및 학습 로그

## 학습법 변천

### 학습법 v1
[강의를 먼저 보는 편? 코드부터 쓰는 편?]

저는 우선 강의를 보기보다는 코드부터 쓰는 편이었던 것 같습니다. 강의를 보는 시간에 일단 코드부터 써보면서 필요하면 주변에 물어보거나 찾아보는 형식으로 진행했습니다.

[AI에 먼저 물어보는 편? 문서를 찾아 읽는 편?]

레벨1 이전에는 AI를 많이 활용했고, 많이 의존했습니다. 레벨1에서는 AI를 절제하려고 노력했습니다. 주변 크루들 토론하며 학습하려고 노력했습니다. 다만, 캠퍼스 일과 시간 이후 홀로 학습할 때는 여전히 AI에 의존하여 해답을 구하는 경향이 남아 있습니다.

[언제 막히고, 어떻게 해결했는가?]

주로 리뷰어의 피드백을 반영하는 리팩터링 단계에서 '왜 이 방식이 더 좋은가?'에 대한 의문이 생길 때 벽을 느꼈습니다. 이를 해결하기 위해 동료들과 머리를 맞대고 토론하며 맥락을 파악하려 노력했고, 집단 지성으로도 풀리지 않는 경우 AI에게 물어봐서 정답을 얻는 방식으로 학습했습니다.

### 학습법 v2
어떤 것을 학습해야할 때 너무 깊게 학습하지말고 일단 AI를 통해 핵심 내용들 훑어보는 식으로 학습한 후 코드에 직접 적용해본다. 거기서 틀리거나 실패하는 상황이 발생할 때 그 부분에 대해 다시 공식 문서, 블로그를 찾아보거나 AI의 도움을 받아 학습한다. 이것들을 계속 반복한다.

### 학습법 v3
1. 핵심 내용 훑기
2. 코드 적용/ 백지 복기→ 실패 경험
3. 공식 문서/AI로 보완

#### 구체적인 실행 방법

학습 내용을 공식 문서를 빠르게 훑어본다. 이후 코드에 적용해보거나 빈 종이에 초등학생도 이해할 수 있는 수준으로 배운 것을 정리해본다. 거기서 실패하는 부분을 찾고 그 부분에 대해 공식 문서나 AI를 통해 학습한다. 이 과정을 반복한다.

#### 해당 내용을 의식하면서 진행하자.
1. 나는 내가 설명할 수 없는 것은 이해하지 못한 것이다.
2. 백지 복기 시 시간 제한을 두자.
    - 5~10분만 정하고 집중해서 적자. 다 적지 못해도 괜찮다. 생각이 나지 않아 쩔쩔매는 순간에 장기 기억으로 넘어갈 준비를 하고 있는 것이다.
3. 학습하다가 다른 개념에 대해 학습이 필요하다고 느낀다면 그 개념은 잠시 미뤄두고 나중에 학습하자.
    - 반드시 기록해놓자.

## 동료 코칭

### 아이디어
**대길 크루**
- 학습 시간이 오래걸리고 백지 복기에서 시간을 줄인다고 했으니, 학습 시간을 줄이고 백지 복기 시간을 더 가져가봐라
- 스프링 학습 테스트 할 때 아는 내용이라 성공해서 바로 넘어갔다고 했는데, 의도적으로 실패를 해봐라

**요크 크루**
- 이해했다고 생각한 부분에서 멈추지 말고 어떻게든 질문을 이끌어내는 방식을 적용해봐라

### 적용 결과
- 대길 크루의 코칭은 아직 적용해보지 못했다.
- 요크 크루의 코칭을 통해 AI의 지침 사항에 학습 시킨 후 해당 내용에 질문을 함으로써 백지 복기를 유도해달라고 프롬프팅하였다.
- 그 결과, 백지 복기를 의식하지 않아도 백지 복기를 수행하게 되어 도움이 되었다.
