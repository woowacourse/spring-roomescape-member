## 학습법
### v1
1단계: 실행 중심의 문제 탐색
코드를 작성하며 맞닥뜨린 기술적 장애물을 AI 질문과 검색을 통해 해결한다.  
이때 AI에게 직접적인 코드 작성을 요청하지 않고 논리적인 해결 방향과 구조에 대해서만 질문하며, 모르는 개념은 핵심 키워드 위주로 파악하여 실제 코드에 직접 적용해 본다.

2단계: ai를 통한 논리 검증
전체 코드에 대해 AI가 제안하는 리팩토링 포인트를 확인하며 설계의 타당성을 검토한다.  
제안된 내용을 무조건 수용하기보다 왜 이 방식이 더 나은가?, 내가 세운 우선순위와 부합하는가? 등라는 의문을 끊임없이 던지는 토론을 통해 개선안을 도출한다.

3단계: 관점 확장을 통한 교차 검증
크루들과의 대화를 통해 내가 미처 발견하지 못한 문제를 발굴하고 다른 관점의 접근 방식을 탐색한다.  
타인의 고민과 해결책을 단순히 정답으로 받아들이지 않고 내 설계 기준과 대조해 보며, 정답이 없는 모호한 상황에서 스스로 가설을 세우고 검증하는 연습을 반복한다.

### v2
1단계: 전체 맥락 파악
먼저 공식 문서를 가볍게 훑어보며 전체적인 구조와 핵심 키워드를 파악한다. 처음부터 모든 내용을 완벽히 이해하지 않고 대략적인 키워드만 파악한다.

2단계: AI 기반 '질문-답변' 학습
AI에게 공식 문서 전문을 학습시킨 뒤(한번에 여러 공식 문서 학습), 지식을 인출할 수 있는 환경을 만든다.
AI에게 "이 문서의 핵심 내용을 바탕으로 내가 이해도를 체크할 수 있는 질문 리스트를 만든 후, 정답은 내가 답변한 뒤에 알려줘"라고 요청
3단계: 심화 질문
  작성한 답변을 바탕으로 이해가 부족하거나 더 궁금한 부분은 AI에게 추가 질문을 던져 세부 내용을 파악. 꼬리에 꼬리를 무는 질문을 통해 파악한다. 이때 내가 필요한 적정 기준을 정하는데, 해당 기준은 일단 이번 과제를 기준으로 한다.(단 코드 기반 질문을 제외한다)

4단계: 정보의 교차 검증 및 외부 탐색
AI의 답변이 모호하거나 신뢰도가 떨어진다고 판단될 때는 다음 과정을 거친다.
1. 해당 파트의 공식 문서를 직접 찾아 대조한다.
2. 공식 문서의 설명이 너무 난해할 경우(이해하지 못할 경우), 외부 검색 및 토의를 통해 '공식 문서를 이해하기 위한 수딘'으로 활용한다.

AI에게 완성된 코드를 제시하고 직접적인 리팩토링 안을 묻는 대신, "이 구조에서 [특정 극한 상황]이 발생한다면 어디가 가장 먼저 무너질지 논리적으로 공격해줘"라고 요청.
