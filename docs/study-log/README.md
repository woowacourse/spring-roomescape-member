# 학습법

## v1

주로 개발 서적을 기반으로 학습을 진행했었습니다.
1. 처음부터 순차적으로 쭉 읽기
2. 이해 안 가는 것이 생기면 구글링 or 유튜브에서 관련 강의 찾아보기
3. 책 내용 이외의 궁금한 것 AI와 질답
4. 누군가에게 나만의 언어로 설명해줄 수 있는지 시뮬레이션

## v2

1. 학습을 하며 기술 부채들을 기록하기 
2. 기술 부채를 해결하는 과정을 꼬리에 꼬리를 무는 마인드맵 형태로 기록하며 학습하기
   - 학습 과정
       1. 구글링을 통해 다른 사람들이 정리한 글 읽어보기
       2. 다른 사람의 인사이트를 얻은 후, 공식 문서 확인하기
       3. AI와 질답을 통해 개념 확장하기
       4. 각 기술 부채에 대한 나만의 답변을 한 문장으로 도출하기
          (Optional) 큰 깨달음을 얻은 기술 부채는 Velog에 과정 기록하기

### v1에서 변경된 부분:

v1에서 순차적으로 학습하던 방식에서 비순차적으로 필요한 부분이 생겼을 때 찾아가서 학습하는 방식으로 변경했습니다.

## v3

1. 학습을 하며 기술 부채들을 기록하기
   기술 부채를 기록할 때 답해야 할 사항:
    - 이 기술 부채를 해결함으로써 얻고 싶은 인사이트가 무엇인지 적기
    - 내가 무엇을 모르는지 적기
2. 기술 부채를 해결하는 과정을 꼬리에 꼬리를 무는 마인드맵 형태로 기록하며 학습하기
    - 제한 시간 30분
    - 학습 과정
        1. AI에게 공식문서 링크와 읽어야하는 공식문서 목차를 요청한다.
        2. AI와 깊은 토론을 진행한다.
        3. 각 기술 부채에 대한 나만의 답변을 한 문장으로 도출하기
           (Optional) 큰 깨달음을 얻은 기술 부채는 Velog에 과정 기록하기 

### 주말에 시도할 것: 쌓인 기술 부채 깊게 학습하기: AI를 학습에 완벽하게 활용하는 방법 찾기

적용 시나리오:
1. [언제·어떤 상황에서 적용할 것인가]
미션을 진행하며 쌓인 기술 부채를 주말에 집중해서 학습할 시간을 확보했을 때 적용한다.

2. [그다음 단계는 무엇인가]
쌓인 기술 부채에 대해서 AI를 활용해 공식 문서와 필요한 학습 범위를 파악한다. 이후 질문을 이어가며 개념을 정리하고, 학습 과정을 기록한 뒤 핵심 내용을 한 문장으로 정리한다.

3. [멈출 조건 — 어떤 신호가 보이면 바꾸거나 중단할 것인가]
한 기술 부채 당 30분의 시간 제한을 정하고 시간 내에 핵심 개념을 정리하지 못하거나, 학습 범위가 과도하게 확장된다고 느껴질 경우 학습을 중단한다.
