## 학습법 v1 → v2 → v3 변천

#### [ v1: 아날로그 학습의 한계와 AI의 도입 (레벨 1 초기) ]
- **파이프라인**: `[ 크루에게 질문(아날로그) ] ➔ [ 시간 부족 및 진도 지연 ] ➔ [ 울며 겨자먹기 AI 사용 ]`
- **회고**: LMS+에는 작성하지 못했지만 초기에는 가급적 AI를 배제하고 크루들과 아날로그 방식으로 소통하며 학습하려 
했습니다. 크루들과 교류하며 함께 성장하는 경험은 너무 좋았습니다.
하지만 배워야 할 지식은 방대했고, 시간적 압박 속에서 이 방식은 속도가 느리다고 느꼈습니다.
현실적인 시간 단축과 빠른 지식 습득을
위해, 결국 울며 겨자먹기로 AI를 사용하게 되었습니다.

#### [ v2: AI 주도의 뼈대 구축과 한계 인식 (레벨 1 후반 ~ 레벨 2 진입 전) ]
- **파이프라인**: `[ AI와 대화하며 뼈대 잡기 ] ➔ [ 스스로 고민하며 구현하기 ] ➔ [ AI에게 피드백받아 다듬기 ] ➔ [ 완성된 인사이트를 크루들에게 내 언어로 설명하기 ]`
- **회고**: 학습 속도를 최고조로 끌어올리기 위해,  우선 AI에게 전체 아키텍처와 초기 뼈대를 
요청했습니다. 구현은 빠르게 진행되었지만 치명적인 부작용이 있었습니다. 낯선 문제 앞에서 스스로 백지상태에서 고민하고
부딪히는 과정이 생략되다 보니, 응용력이 현저히 떨어졌습니다. 코드가 내 머릿속 지식으로 남지 않고 수동적인
학습에 머무르고 있음을 깨달았습니다.

#### [ v3: 일단 아는 것을 바탕으로 최대한 지식을 짜 내어보고 검증받자! (현재 레벨 2 진행 방식) ]
- **파이프라인**: `[ 스스로 개념 유추 및 백지 가설 세우기 ] ➔ [  AI에게 내 논리 피드백 받기] ➔ [구현 시도] ➔ [AI에게 내 논리 피드백 받기] ➔ [크루들과 지식 공유하기]`
- **회고**: v2의 응용력 부재를 극복하기 위해, AI의 사용 방식을 완전히 뒤집었습니다. 처음부터 AI에게 정답이나 뼈대를 
받지 않습니다. 모르는 개념이 나오더라도 일단 제가 아는 지식을 총동원해 있는 그대로 유추해 보고, 스스로 로직의 뼈대를
세워봅니다. AI는 정답 생성기가 아니라 "내가 이런 구조로 접근했는데, 이 논리가 맞아?"라고 묻는 '논리 검증기'로만
사용합니다. 이를 통해 속도와 스스로 생각하는 '설계 근육'의 밸런스를 맞춰나갈 것입니다. 최종으로는 확립한 제 지식을
크루들에게 공유해보며 Output을 확인합니다. 이 때 설명을 잘 못하는부분을 다듬어 다시 학습할 수 있고 크루들의 다양한 의견을
비교해보며 날카롭게 유지할 수 있을것이라 생각합니다.

주말에 시도할 것: 키워드 작성을 이용한 마인드맵 그려보기

적용 시나리오:
1. 리팩토링을 할 때 막힌 부분이 있으면 그와 관련된 키워드를 나열한다.
2. 키워드를 바탕으로 몰랐던 부분을 학습하며 가지치기 방식으로 공부를 해나간다.
3. 하다가 너무 시간이 오래 걸린다고 판단되거나 생각보다 정리가 잘 되지 않을 때 그만둔다.
