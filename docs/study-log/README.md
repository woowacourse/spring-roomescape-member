## 주말에 시도할 것
소크라테스 AI를 활용할 때 타임박싱을 적용한다. (Feedback Loop 강화)

적용 시나리오:
1. **문제 탐색**: 금요일 오후에, 내 방탈출 예약 애플리케이션의 문제점을 찾는다.
2. **타임박싱을 적용한다**: 문제 정의 15분, 문제 해결 15분으로 시간을 제한한다.
3. **문제 정의 15분**: 외부 지식(AI, 구글링 등)에 아예 의존하지 않고, 스스로 생각하며 문제를 정의하고 개선책을 제시한다.
4. **문제 해결 15분**: 내가 정의한 문제와 제시한 해결책이 바람직한지 소크라테스 AI와 논의하고, 코드에 반영한다.
5. **중단**: 타임박싱 시간을 초과해서도 해결할 수 없으면 멈추고, 다른 크루의 도움을 받거나 AI에게 해결책을 직접 물어본다.

## 학습법 변화 과정

### 학습법 v1

#### 학습법 전문
정답이 없는 문제(객체지향, 클린코드)를 만났을 때
- 내 머리속에 떠오른 방법이 있다면, 우선 코드부터 작성해본다.
   이후, 주변 크루에게 기회가 될 때 서로의 구현을 공유하고 트레이드 오프를 분석하는 시간을 갖는다.
- 어떤 방법을 선택해야 할지 모르겠다면, 페어 혹은 주변 크루에게 이 문제를 어떻게 해결했는지(혹은 해결할건지)를 물어보고, 서로의 의견의 장단점을 분석하고 이야기해본다.
- 의견을 나눌만한 크루가 없다면 AI에게 물어본다.

비교적 정답이 있는 문제를 만났을 때
- AI에 먼저 물어본다.
- AI가 제시한 답에서 느껴지는 문제점을 반문하길 반복해, 만족스러운 답안을 찾아낸다.

학습을 마무리할 때
- 학습 도중 큰 깨달음을 얻었을 때 마다 메모장 등에 간략하게 정리해놓는다.
- 미션을 마치고 나면, 해당 정리본을 기반으로 노션에 학습 회고록을 자세하게 작성한다.
- 이 과정에서 받았던 피드백 및 깨달음을 나의 언어로 다시 정리하는 시간을 갖는다.

<br>

### 학습법 v2

#### v1의 문제점 & v2의 변화
미션 도중 얻은 깨달음을 서로 연결지어 기록하기 어려웠고, 기록하지 않은 깨달음은 모두 휘발되었다.
- 마인드맵을 도입해, 지식을 모두 기록하면서도, 지식 간에 연결고리를 만든다.

AI가 정답을 너무 빨리 알려주어, 깊게 고민할 수 없었다.
- AI가 내가 고민하는 것을 도와주도록, 소크라테스 대화법을 적용한다.

#### 학습법 전문
미션 도중 얻은 깨달음을 사소한 것도 기록하되, 다음 규칙을 따른다.

1. 나중에 봐도 떠오를 수 있다는 전제 하에, 최대한 간결하게 작성한다.
2. 마인드맵 형태로 작성해, 지식을 서로 연결한다.

미션 도중 소크라테스 AI를 주기적으로 사용해 내 코드를 검토해 피드백받는다.

1. AI가 답을 말하지 못하게 제한한다.
2. AI의 답변을 비판적으로 생각하고, 내 생각을 질답하는 것으로 깊이 있게 피드백한다.
3. 정답이 있는 문제일 경우, 소크라테스 AI 대신 일반 AI를 사용한다.
   내가 학습한 내용을 설명하고, 잘못 이해한 부분이 있는지 피드백받는다.

학습을 완료하면 블로그에 회고록을 작성하는 것으로 마무리한다.

1. 다른 사람이 읽었을 때 이해하기 쉽게 작성하려 노력한다.
2. 시간 여유가 있을 경우 최대한 빠른 시일 내에 작성한다.

<br>

### 학습법 v3

#### v2의 문제점 & v3의 변화
미션 진행과 마인드맵 작성을 동시에 병행하기 어려웠다.
- '지식' 자체를 기록하지 않고, "내가 어떤 상황에 왜 그 선택을 했는지"를 기록한다.
- 이를 통해, 자세하게 기록하지 않아도 이후 회고에 활용할 수 있도록 만든다.

쉬지 않고 계속 다음 내용을 학습하니, 어느 순간 집중하기 힘들다.
- 학습 도중에 의식하여 쉬는 시간을 갖는다.
- 쉬는 시간 동안 산책하여, 방금 학습한 내용의 핵심을 머리속으로 그려 본다.

#### 학습법 전문
미션 도중 얻은 **깨달음을 기록**하되, 다음 규칙을 따른다.

1. 고민했던 의사결정의 지점을 메모장에 3-Step 앵커로 메모한다.
    - 3-Step 앵커란?
        1. 상황(Trigger)
            - 직면한 문제나 선택지를 의미한다.
            - ex) “DB 동시성 제어 필요”
        2. 결정(Action)
            - 내가 내린 선택을 의미한다.
            - ex) “낙관적 락 채택”
        3. 근거(Reason)
            - 왜 그렇게 생각했는지를 의미한다.
            - ex) “충돌 빈도 낮음 + 성능 저하 방지 우선”
2. 미션이 끝난 후 이 앵커들을 마인드맵의 노드로 연결한다.
    - 마인드맵엔 추가되지만 당장 파고들고 싶지 않은 개념이 있다면, 해당 노드의 투명도를 낮춰 나중에 학습할 것임을 표시한다.
    - 나중에 해당 개념을 정말 학습해야 할 때 학습한다.
      가능하다면, 해당 개념을 나중에 학습해야겠다고 느낀 이유를 같이 적는다.

미션 도중 소크라테스 **AI**를 주기적으로 사용해 내 코드를 **검토**해 피드백받는다.

1. AI가 답을 말하지 못하게 제한한다.
2. AI의 답변을 비판적으로 생각하고, 내 생각을 질답하는 것으로 깊이 있게 피드백한다.
3. 정답이 있는 문제일 경우, 소크라테스 AI 대신 일반 AI를 사용한다.
   내가 학습한 내용을 설명하고, 잘못 이해한 부분이 있는지 피드백받는다.

학습을 완료하면 블로그에 **회고록**을 작성하는 것으로 마무리한다.

1. 다른 사람이 읽었을 때 이해하기 쉽게 작성하려 노력한다.
2. 시간 여유가 있을 경우 최대한 빠른 시일 내에 작성한다.

학습 중간에 쉴 때, 스마트폰을 보는 대신 **산책**하거나 창밖을 본다.

- 이때, 방금 학습한 내용의 핵심을 머리속으로 그려본다.
  이를 통해 각 지식 간의 고리를 연결한다.
