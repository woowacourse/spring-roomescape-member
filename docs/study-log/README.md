## 학습법 변화 과정

### 학습법 v1

[제출 내용]

``` shell
주로 인사이트를 얻는 것에 집중하며 학습을 해왔다. 기존에 몰랐던 지식, 새로 알게된 지식, 연결되는 지식 등을 캐치하려고 하는 편이었다.
학습할 때는 주로 AI를 활용한다. 개념과 간단한 사용법을 알아봤던 거 같다. 그리고 직접 적용해본다.
궁금증이 생기면 따로 정리한다. (예: DriverManager.getConnection()을 수행하면 내부적으로 어떻게 동작할까?)
나 혼자 궁금증을 도출하는 경우도 있지만, 크루와 대화 또는 리뷰어와 대화에서 많은 궁금증을 도출하는 거 같기도 하다.

궁금증을 파헤칠 떄는 주로 공식 문서나 라이브러리를 직접 찾아보려고 한다.
라이브러리 내부 코드가 이해하기 어려울 땐, AI에게 "내가 DriverManager.getConnection()이 내부적으로 어떻게 동작하는지 확인해보려고 하는데, 라이브러리 코드에서 실제 커넥션이 맺어지는 부분은 어디야?"와 같이 묻고, 조금씩 이해해보려고 했다.
그리고 학습한 결과를 나만의 언어로 한 문장으로 정리한다.
```

```shell
두 가지 단계로 나뉜다. 해당 단계는 "시간을 준수하기 위한 학습법"이다.

## 기술 부채 쌓기
### 지식이 명시적이지 않을 때 접근 방법
*목적: 내가 모르는 게 뭔지 아는 걸 우선으로 한다*
1. 상황을 한 문장으로 정리한다.
	예) 스프링에서 HTTP 통신은 어떻게 해야하지?
2. AI를 사용해 선택지와 방법을 물어 큰 그림을 알아낸다.
3. 명시적 접근 방법을 적용한다.

### 지식이 명시적일 때 접근 방법
*목적: 일단 사용법만 알고 넘어가는 걸 우선으로 한다.*
1. 궁금증이 생긴 주체의 역할을 간단히 유추한다.
	예) DriverManager.getConnection()이 뭐지? 커넥션?? 뭔가 연결해 주는 건가?
2. 메타정보를 파악하여 지금 내가 사용해야 할 지식인지 파악한다.
3. AI를 통해 해당 주체가 "어떤 맥락에서 사용되는지"를 한 줄로 정리한다.
	예) "DriverManager.getConnection()은 외부 DB 서버로부터 DB 커넥션을 얻어올 때 사용한다."
4. 추가로 생기는 궁금증은 모두 "기술 부채" 항목에 추가한다.

## 기술 부채 청산
1. 일주일 내에 적용할 수 있는 기술 부채를 찾는다.
2. 옵시디언 마인드 맵을 이용해 정보를 찾아나간다.
	1. 발산되는 궁금증들도 함께 뻗어나간다.
	2. 기술적 깊이보단, 기술의 철학이나 해결 목표를 기준으로 바라보려고 노력해본다.(해당 단계는 아직 추상적. 구체화 필요)
	3. 깊이는 3을 넘지 않도록 한다.
```

### 최종 학습 전략

- [구현 모드] 목표를 달성하기 위한 최소한의 정보만을 학습하여 적용하고, 그 이상의 정보는 기술 부채 로그에 쌓아둔다.
    - [필요 지식 탐색] AI를 이용해 정보를 묻되 공식 문서로부터 참조한 정보로만 학습한다.
    - [고민 포인트 의식하기] 팔짱을 끼거나 허공을 응시하고 있다면 "무엇때문에 고민중인가?"를 의식하여 기록해본다.
- [학습 모드] 쌓인 기술 부채에서 필요한 지식을 골라 학습한다.
    - [학습 전략] 다음 선택지를 고려한다:
        - Claude Code로 전용 학습 테스트 레포지터리 생성 및 수행
        - 마인드 맵 작성(유기적 연결 및 관계 확인)
        - AI에게 질문(공식 문서 기반) 및 실제 공식 문서 확인하고 정리하기
    - [종료 기준] 맥락과 흐름을 자신의 언어로 정리한다. (크루에게 설명하면 더 좋음)

### 느낀점

레벨 1이 끝날 무렵부터 학습 방법과 기록 방법에 대한 고민이 정말 많았다.
방학동안 나에게 맞는 학습 방법이 뭔지, 우테코에서 얻어가고 싶은게 뭔지, 취업을 어떻게 대비해야 할지, 개발자를 하고 싶은 게 맞을지.. 삶에 대한 방향성을 비롯한 많은 고민을 했다.

그런데 마침 레벨2 에서 "나만의 학습법 발견하기"가 주제였어서 반가운 마음이 있었다.
나름 몇 주간 고민해왔기에 나만의 학습 전략이 있다고 생각했는데, 이번 과정에서 사실은 제대로 실천도 못하고 기준도 모호했음을 알게되었다.

돌아보는 과정에서 "필요성을 느끼고 학습한 것"과 "직접 파헤쳐본 것"이 오래 기억에 남았다는 점을 발견했고, "필요성을 느끼면 그때 알아본다"라는 기준을 잡게되었다.

학습법을 지키려고 노력하는 것은 생각보다 불편한 작업이었지만, 그 덕분에 정말 실천할 수 있는 학습법이 남았던 거 같다.
