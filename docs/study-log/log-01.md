### 2. 이번 타임의 학습 전략

- 이전에 바꾸기로 한 전략은 무엇이었고, 실행했는가?
    - AI에게 도움을 청해 학습하고자 하는 개념의 학습단위를 작게 쪼개고,
      20분 내에 백지에 복습하거나 코드를 작성하는 시간 제한 기반의 시스템을 설정했다.
    - 이때 학습 자료는 최대한 공식 문서를 사용한다.
    - 이번 미션에서 실제로 AI에게 15~20분 단위로 목차를 쪼개달라고 요청했고,
      타이머를 맞추고 그 시간 내에 학습, 복습, AI로 복습 내용 피드백하기의 과정으로 학습을 진행했다.

- 실제로 어떻게 학습했는지 디테일한 과정을 써보세요.
    - 미션1에 제시된 개념들 중 내가 모르겠는 개념(어노테이션, 스프링이란, CRUD, API란, gradle 등)을
      학습하기 위해서 AI에게 도움을 청했다.
      각 개념을 학습할 수 있도록 최대한 공식문서를 제공해주고, 각 개념 학습에서 얻어가야할 포인트를 집어달라고 했다.
    - AI가 제시해 준 학습 포인트를 위주로 20분 동안 공식 문서를 보며 각 개념을 학습 했다.
    - 학습한 개념을 제대로 이해했는지 확인하기 위해서 백지에 이해한 개념을 작성하거나 코드로 작성해보았다.
    - LMS에 제공된 학습테스트로 MVC와 CRUD API 학습을 진행했다.
        - 학습테스트에 제공된 설명을 읽고, 같이 첨부된 공식문서 참고자료에 들어갔다.
        - AI에 각 학습테스트에서 배워야할 내용과 공식문서를 넣어서 학습에 필요한 내용을 요청했다.
          제공된 공식문서에서 어디를 중점적으로 학습해야 하고, 공식문서를 읽고 어느 정도까지 학습해야 하는지 요청했다.
            - 이 때도 위와 같이 시간을 정해서 학습하고, 복습하고, AI를 사용해 피드백하는 방식으로 학습을 진행했다.

### 3. 전략 평가

- 효과적이었던 것과 그 이유
    - 학습 단위 쪼개기
        - AI를 사용해 방대한 공식 문서 중 '지금 당장 봐야 할 곳'을 필터링하고작게 쪼갰더니 어디서부터 얼만큼 공부해야 하는지에 대한 막연함이 줄었다.
        - 학습을 시작하고, 진행하는 데 심리적 장벽이 낮아졌다.

- 비효과적이었던 것과 그 이유
    - 시간을 제한해서 학습하기
        - 20분이라는 시간이 어떤 개념에는 너무 길고, 어떤 개념에는 너무 짧았다.
        - 특히 코드를 짜보며 학습할 때는 흐름이 끊겨서 오히려 효율이 떨어지기도 했다.
        - 그렇다보니 임의로 20분을 넘는 학습 시간을 가져간 때가 있었는데, 이 때 얼마나 더 공부할 지 계획 없이
        - 무작정 공부하게 되고, 갑자기 한 개념에 너무 많은 시간을 들이게 되는 경우도 있었다.
        - 또 이렇게 시간에 치이다 보니 학습한 개념을 셀프 피드백하는 과정을 건너뛰거나 대충 하게 되었다.

    - 공식 문서 중심 학습의 한계
        - 공식 문서는 개념의 정의에는 강하지만, 실제 코드 개념이 어떻게 적용되는지 보여주는 예시가 부족하다.
          이로 인해 이론과 실전 코드를 연결하는 데 너무 많은 에너지와 시간을 사용했다.
          이 에너지와 시간의 사용이 학습에 도움이 되지 않는다고 느꼈다. 특히 아예 모르는 개념을 학습하고 코드로 옮길 때 더욱 그런 것 같다.

- 막힌 것의 종류(1번)와 전략의 궁합은 어땠는가?
    - 모르는 개념을 AI가 정해준 학습 단위별로 학습하니 어떻게 학습을 시작하기 좋았고, 실제 학습에 도움이 되었다.
    - 하지만 실제로 코드로 작성하는 것에는 궁합이 좋지 않았다.

### 4. AI 피드백

- "피드백의 단위를 '포스트잇' 수준으로 낮추세요."
  시간이 부족할 때 거창한 백지 복습은 오히려 독이 됩니다. 앞으로는 코드를 짤 때 [기능 하나 완성 -> 1분 셀프 문답 -> 커밋]이라는 아주 짧은 루프를 돌려보세요.
  거창한 문서화 대신, 내가 방금 쓴 코드 옆에 // 왜 이렇게 짰지?에 대한 답을 주석 한 줄로 남기는 것만으로도 충분히 훌륭한 셀프 피드백이 됩니다.

### 5. 다음 타임에 바꿀 것

- 유지할 것과 그 이유
    - AI를 통한 학습 단위 쪼개기: 방대한 공식 문서에서 길을 잃지 않게 해주므로 계속 유지한다.
      덕분에 심리적 장벽이 낮아지는 효과를 톡톡히 봤다.
- 바꿀 것과 그 이유
    - 고정된 20분 타이머 폐지: 대신 학습 주기를 변경한다.
      한 기능을 완성하는 것을 하나의 학습 단위로 보고, 기능이 끝나는 시점에 복습을 진행하여 학습 효율을 높인다.
    - 공식 문서 + 예시 코드 병행: 아예 모르는 개념은 문서부터 파지 않고, AI에게 '우리 미션 상황에 맞는 코드 예시'를 먼저 요청하여 구조를 눈에 익힌 뒤 문서를
      참조하는 '역방향 학습'을 도입하겠다. (이론-실전 연결 에너지 절약)
    - '1분 셀프 피드백' 습관화: 백지 복습이 부담스러울 땐, 커밋 직전 "이 코드를 왜 이렇게 짰는가?"에 대해 스스로 1분만 자문자답하고
      그 내용을 커밋 메시지나 주석에 짧게 남기는 방식으로 피드백 루프를 가볍게 유지하겠다.
