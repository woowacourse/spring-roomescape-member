```text
## 목표
@Controller와 @RestController의 차이점을 이해한다.

## 학습 이유
Controller를 만들 때 @Controller 어노테이션을 사용해야 할지, @RestController 어노테이션을 사용해야 할지 고민이 들었다. 

## 정보 수집
- 공식 문서 - @Controller: https://docs.spring.io/spring-framework/docs/current/javadoc-api/org/springframework/stereotype/Controller.html
	- 해당 클래스가 Controller임을 나타낸다. 보통 @RequestMapping 어노테이션을 기반으로 하는 핸들러 메서드들과 함께 사용된다.

- @RequestMapping이란?
	- 특정 주소(URL)로 들어온 요청을 어떤 메서드가 처리할지 연결하기 위해 사용하는 어노테이션

- 공식 문서 - @RestController: https://docs.spring.io/spring-framework/docs/current/javadoc-api/org/springframework/web/bind/annotation/RestController.html
	- @Controller와 @ResponseBody가 결합된 편의용 어노테이션이다. 이 어노테이션이 붙은 타임은 컨트롤러로 취급되며, @RequestMapping이 붙은 메서드들은 기본적으로 @ResponseBody의 의미를 내포하게 된다. 

- @ResponseBody란?
	- 자바 객체를 HTTP 응답 본문의 데이터로 변환하여 클라이언트(브라우저나 앱)에 직접 전달할 때 사용하는 어노테이션
	- 객체를 JSON으로 변환해 응답

- @RestController에서 @RequestMapping이 붙은 메서드는 왜 @ResponseBody의 의미를 내포하는가?
	- @ResponseBody 어노테이션이 @RestController 클래스 레벨에 붙어있기 때문에 전체 메서드에 적용
	
## 나중에 더 알아볼 것 (깊이 제한)
- ResponseBody는 객체를 JSON으로 변환해서 어디에, 왜 응답하는 것인지?

## 나만의 언어 (구체적 예시 포함)
@Controller는 해당 클래스가 컨트롤러임을 표현하기 위한 어노테이션이다. 
기본적으로 @Controller는 View를 직접 반환하고, 데이터만 반환하고 싶으면 메서드에 @ResponseBody 어노테이션을 추가해야 한다. 
@RestController는 @Controller와 @ResponseBody가 결합된 어노테이션이다.
@RestController를 사용하면 모든 메서드에 자동으로 @ResponseBody 어노테이션이 상속되기 때문에, 따로 어노테이션을 추가하지 않아도 돼서 간편하다. 

@Controller는 기본적으로 가구의 완성품을 보낸다면, @RestController는 가구의 부품만 보낸다. 

## 나만의 언어 검증
- (오개념) @Controller가 HTML 파일을 직접 반환하는 것이 아니다. View의 이름을 반환하는 것이다. 
- (오개념) 어노테이션은 상속보다는 메타 어노테이션이 전파되는 개념이다. 
	- 메타 어노테이션은 어노테이션을 만들 때, 그 어노테이션의 성격이나 동작을 정의하기 위해 위에 붙이는 어노테이션을 의미
	
## 결론
@Controller는 해당 클래스가 컨트롤러임을 표현하기 위한 어노테이션이다. 
기본적으로 @Controller는 View의 이름을, 데이터만 반환하고 싶으면 메서드에 @ResponseBody 어노테이션을 추가해야 한다. 
@RestController는 @Controller와 @ResponseBody가 결합된 어노테이션이다.
@RestController를 사용하면 모든 메서드에 자동으로 @ResponseBody 어노테이션이 적용되기 때문에, 따로 어노테이션을 추가하지 않아도 돼서 간편하다. 

@Controller는 기본적으로 가구의 완성품을 보낸다면, @RestController는 가구의 부품만 보낸다. 
```
