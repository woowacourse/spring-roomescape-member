## 학습 로그 #1

**시간**: 04/28 12:00 ~ 12:45 (약 45분)
**학습 범위**: 1단계 MVC

### 1. 막힌 것의 종류
이번에 막힌 것은 어떤 종류의 어려움이었는가? (해당하는 것에 체크)
- [ ] 개념 자체를 모르겠다 (예: "스프링 빈이 뭔지 모르겠다")
- [ ] 개념은 알겠는데 코드로 어떻게 쓰는지 모르겠다 (예: "JdbcTemplate 문법을 모르겠다")
- [ ] 코드는 돌아가는데 이게 맞는 건지 모르겠다 (예: "계층 분리를 이렇게 해도 되나?")
- [x] 기타: 기존에 사용하지 않는 방식을 사용해 낯설었다.

### 2. 이번 타임의 학습 전략
- 이전에 바꾸기로 한 전략은 무엇이었고, 실행했는가?
  - 미션에서 제시한 학습 테스트를 먼저 풀이했고, 이 학습 테스트에서 배운 내용을 기반으로 미션을 수행했습니다.
  - 이 과정에서 AI 사용을 지양하고, 제공된 학습 자료만 활용해봤습니다.
- 실제로 어떻게 학습했는지 디테일한 과정을 써보세요.
  1. 제공된 학습 테스트(MVC Response, CRUD API)를 풀이한다.
  2. 풀이 과정에서 어려운 내용이 있다면, 최소 3분 정도 혼자서 고민해본다.
  3. 혼자 고민해서 해결할 수 없었다면, 학습 테스트의 답안(complete)을 참고해 해결법을 찾는다.
  4. 학습 테스트를 모두 풀이한 후, 미션을 구현한다.
  5. 구현 중 막히는 것이 생긴다면, 내가 풀이한 학습 테스트 코드를 참고해 해결법을 찾는다.

### 3. 전략 평가
- 효과적이었던 것과 그 이유
  - 학습 테스트를 통해 새로운 기술을 학습한 것
  - 새로운 기술에 대한 모범 사례를 빠르게 학습할 수 있어, 도입이 쉬웠습니다.
- 비효과적이었던 것과 그 이유
  - AI 사용을 아예 배제한 것
  - 학습 테스트에서 제공해주지 않은 내용 중 명확한 정답이 없는 문제(메서드의 네이밍 등)를 해결하기 어려웠습니다.
- 막힌 것의 종류(1번)와 전략의 궁합은 어땠는가?
  - 좋은 튜토리얼을 통해 새로운 기술을 학습한 것은 효과적이었습니다. 해당 기술을 어떻게 사용하면 좋을지에 대한 모범 사례를 단기간에 학습할 수 있었습니다.
  - 하지만 튜토리얼에 담겨 있지 않은 내용이 문제가 되었을 때, 좋은 튜토리얼을 또 탐색하는데 시간이 오래 걸릴 것이라 추측합니다. (이번에는 학습 테스트를 참고하는 것으로 한정지었으니, 별도의 튜토리얼을 탐색하지 않았습니다.)

### 4. AI 피드백
- 자신의 학습 전략에 대해 AI 학습 전문가에게 피드백을 요청하고,
  유용했던 제안 1가지 이상 기록
  - 한정된 분야(네이밍, 컨벤션, 추상화 수준 등)에 대해서만 AI를 활용한다.
  - AI를 활용하되, 정답으로 수용하지 않고 비판적인 검토 대상으로 대우한다.

### 5. 다음 타임에 바꿀 것
- 유지할 것과 그 이유
  - 새로운 기술을 사용할 때, 학습 테스트(튜토리얼 문서)를 우선 확인한다.
  - 해당 기술이 등장한 배경과 철학을 단기간에 학습할 수 있어, "왜 사용하는가?"라는 궁금증을 빠르게 해결할 수 있습니다.
  - 이 덕에, 해당 기술을 의도에 맞게 사용할 수 있습니다.
- 바꿀 것과 그 이유
  - AI 사용을 지양한다.
  - 좋은 튜토리얼을 탐색하되, 튜토리얼에서 다루지 않은 궁금증은 AI를 통해 탐구해보는 것이 학습 속도를 훨씬 가속할 것이라 판단합니다.
