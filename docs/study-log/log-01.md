## 학습 로그 #01

- **시간**: 04/28 16:30 ~ 17:24 (약 54분)
- **학습 범위**: 1단계 스프링 문법

### 1. 막힌 것의 종류
이번에 막힌 것은 어떤 종류의 어려움이었는가? (해당하는 것에 체크)
- [ ] 개념 자체를 모르겠다 (예: "스프링 빈이 뭔지 모르겠다")
- [ ] 개념은 알겠는데 코드로 어떻게 쓰는지 모르겠다 (예: "JdbcTemplate 문법을 모르겠다")
- [ ] 코드는 돌아가는데 이게 맞는 건지 모르겠다 (예: "계층 분리를 이렇게 해도 되나?")
- [x] 기타: 스프링을 간만에 다뤘기에 문법을 까먹은 부분이 존재했다.

### 2. 이번 타임의 학습 전략

- 이전에 바꾸기로 한 전략은 무엇이었고, 실행했는가? 
  - 1단계라 바꾸지 않았습니다.
- 실제로 어떻게 학습했는지 디테일한 과정을 써보세요.

학습 주제나 키워드는 `모르는 것 / 잊어버린 것 / 알고는 있지만 설명하지 못하는 것`으로 구분했습니다.

아무래도 오랜만에 스프링을 다시 써보는 터라 `잊어버린 것`에 해당하는 내용들이 많았는데요.
ex) `@RequestBody`, `@PathVariable` ...

잊어버린 것은 해당 키워드를 이전에 학습했던 소스를 찾아 읽어보며 기억을 되살렸습니다.
스프링 문법은 정리해둔 코드와 필기를 참고했는데 
미션을 진행하니 실제 코드에 적용하는 것까지를 학습 단위로 잡히며 학습 계획과 자연스레 어우러졌습니다.

### 3. 전략 평가
- 효과적이었던 것과 그 이유
  - 잊어버린 것에 이전에 학습했던 소스를 찾으니 기억이 살아났습니다.
  - 정리된 자료가 있기에 빠르게 학습/적용할 수 있었습니다.
  - 자료와는 다른 코드에 적용해봄으로서 보다 기억에 남았습니다.
- 비효과적이었던 것과 그 이유
  - 아직 1단계라 다른 학습 전략을 적용해보기엔 적합치 않았습니다.
- 막힌 것의 종류(1번)와 전략의 궁합은 어땠는가?
  - 짧은 키워드와 이전에 알던 내용이라 빠르게 진행할 수 있어서 궁합이 좋았습니다.

### 4. AI 피드백
- 자신의 학습 전략에 대해 AI 학습 전문가에게 피드백을 요청하고,
  유용했던 제안 1가지 이상 기록
  - 정리한 개념을 실제 코드나 설계에 적용해보는 것. 이라는 조언이 좋았습니다.
  물론 미션을 진행하며 자연스레 적용이 된 사례이지만, 그럼에도 되살아난 기억을 장기기억으로 전환하는데 탁월한 방법이라 생각합니다.

### 5. 다음 타임에 바꿀 것
- 유지할 것과 그 이유
  - 기존 학습 전략을 유지하겠습니다.
- 바꿀 것과 그 이유
  - 단계 진행에 따라 필요한 학습이 다를 것 같은데, 아직은 바꾸지 않겠습니다.
