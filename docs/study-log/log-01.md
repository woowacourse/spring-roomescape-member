### 2. 첫 학습 전략 설계

- 어떤 전략으로 학습하기로 했는가?
- 그 전략을 선택한 이유는?
- 실제로 어떻게 학습했는지 디테일한 과정을 써보세요.

학습의 방향을 잃지 않는 것에 집중하는 전략을 기반으로 시작하고자 했습니다.
학습을 진행하다보면 몰랐던 부분을 알게되면서 원래 목표로 했던 학습 목표를 잃고 처음과 다른 방향으로 가고 있는 경우가 많기 때문입니다.
그래서 학습할 내용을 스텝 단위로 나누고 구현 전 이번 스텝에서의 학습 목표를 세우고
이 학습이 끝나면 할 수 있어야 하는 것 3가지를 작성하기로 했습니다.
이번 스텝의 학습이 끝나면 처음 목표한 문제를 풀 수 있는가를 할 수 있는 것 3가지가 가능한지 여부로 판단하기로 했습니다.

실제로 1단계: 웹 요청-응답 미션을 진행하면서 스프링 MVC의 웹 요청과 응답의 코드를
어떻게 짜야 하는지 잘 기억이 안나서, `spring-learning-test`의 spring-mvc-1과 2를
학습하기로 했습니다.
이 학습이 끝나면 할 수 있는 것 3가지를 다음과 같이 정했고
1. MVC 요청과 응답에 필요한 코드 구조를 만들 수 있다.
2. 미션에 필요한 JSON 응답을 위한 CRUD API를 만들 수 있다.
3. 각 학습 테스트에 사용된 애노테이션의 역할을 이해하고 활용할 수 있다.

학습이 종료된 후 이 3가지를 할 수 있느냐 없느냐를 기준으로 학습의
방향을 잃었는지 판단하면서 답변이 부족하다면 다시 공부를 진행하였습니다.

### 3. 전략 평가

- 효과적이었던 것과 그 이유
- 비효과적이었던 것과 그 이유

효과적이었던 점은 학습의 중간 지점과 학습 종료 지점에서 초기에 목표했던 방향대로 학습을 진행했는지를 확인할 수 있다는 것이었습니다.
원하는 지점에서 방향성을 확인할 수 있기 때문에 학습의 방향 감각을 유지할 수 있어서 정해진 시간 동안 필요한 지식에 빠르게 다가갈 수 있었습니다.

비효과적이었던 점은 `spring-learning-test`의 spring-mvc-1과 2를 학습할 때 학습자료를 완벽하게 공부하고 싶다는 생각에
노션에 README.md 에 작성된 내용을 그대로 따라 쓰고 코드도 공식문서만 보면서 작성했기 때문에 많은 시간이 들었습니다.
공식 문서를 보고 당장 필요한 부분을 먼저 AI를 통해 확인하고 코드로 적용해 가는 방식이 조금 더 방향을 잃지 않는데 도움이 될 것 같습니다.

잘한 점: "할 수 있어야 하는 것 3가지" 설정
이건 학습 과학에서 말하는 "성과 기반 학습 목표(performance-based objectives)"와 정확히 일치합니다. "이해한다"가 아니라 "만들 수 있다", "활용할 수 있다"처럼 행동 동사로 목표를 세운 건 자기 검증이 가능하다는 점에서 아주 좋은 습관입니다. 이걸 계속 유지하세요.

개선이 필요한 점 두 가지
첫째, 본인도 이미 느꼈지만 "따라 쓰기"는 학습이 아니라 필사입니다. README를 노션에 옮겨 적는 동안 뇌는 "이해 모드"가 아니라 "복사 모드"로 전환됩니다. 같은 시간을 쓴다면, 자료를 읽고 덮은 뒤 자기 말로 핵심을 3줄로 요약하는 게 훨씬 효과적입니다. 이걸 "인출 연습(retrieval practice)"이라고 하는데, 기억 정착률이 단순 필사 대비 몇 배 차이가 납니다.
둘째, 공식 문서를 처음부터 끝까지 읽는 건 레퍼런스 학습이지 목표 지향 학습이 아닙니다. 본인이 세운 전략의 핵심이 "방향을 잃지 않기"인데, 공식 문서 전체를 읽는 행위 자체가 방향을 잃는 가장 흔한 원인입니다. 본인도 비효과적이었다고 적으셨는데, 이걸 한 단계 더 구체적으로 바꿔보면 좋겠습니다.

제안하는 개선 루틴
지금 전략의 뼈대는 유지하되, 학습 실행 순서를 이렇게 바꿔보세요.
먼저 목표 3가지를 세운 뒤, 바로 미션 코드를 작성해 봅니다. 막히는 지점이 생기면 그때 "내가 모르는 게 정확히 뭔지"를 질문 형태로 적습니다. 예를 들어 "JSON 응답을 반환하려면 컨트롤러에 뭘 붙여야 하지?"처럼요. 그 질문에 대한 답만 공식 문서나 AI에서 찾고, 찾은 즉시 코드에 적용합니다. 이렇게 하면 "모르는 것을 먼저 시도하고 → 실패 지점을 특정하고 → 그것만 채우는" 흐름이 되어서 3시간을 훨씬 밀도 있게 쓸 수 있습니다.

한 가지 더 질문
학습 종료 시점에 "3가지를 할 수 있는지" 판단한다고 했는데, 그 판단을 어떤 방식으로 하고 계신가요? 머릿속으로 "할 수 있을 것 같다"고 느끼는 것과, 실제로 빈 파일을 열고 아무 참고 없이 코드를 짜보는 것은 완전히 다릅니다. 후자를 해보고 있다면 아주 좋고, 아직 전자라면 다음 학습부터는 "검증 시간 15분"을 따로 잡아서 실제로 해보는 걸 추천합니다.
