### 2. 이번 타임의 학습 전략

테스트가 실패하는 이유를 찾고, 문제 해결만을 위해 학습하는 전략을 사용했다.

- 학습 과정 예시

```text
## 테스트를 스프링 환경에서 실행시키는 방법: @SpringBootTest
1. 테스트의 jdbcTemplate에서 NullPointerException이 발생했다.
2. 메인에서는 실행해도 에러가 안 나는데, 테스트에서만 에러가 나는 걸 확인했다. 
3. AI를 활용해 현재 테스트 코드에서 NullPointerException이 발생하는 이유를 찾아봤다.
4. 테스트에서 @AutoWired로 JdbcTemplate을 주입받고 있는데, @SpringBootTest 애노테이션을 붙이지 않아 스프링 환경 위에서 실행되지 않는다는 걸 깨달았다.

## 객체를 직렬화할 때는(ResponseEntity 만드는 과정) 해당 객체의 public getter 사용
1. ReservationRequestDto를 따로 만든 후, Reservation에서 getter를 쓸 일이 사라져서 getter를 모두 삭제했다.
2. 정상적으로 통과하던 테스트가 깨지고, 406 에러가 발생했다.
3. 406 에러의 원인을 인터넷 검색으로 찾아봤다.
4. 스프링 부트에서 직렬화를 위해 Jackson 라이브러리를 사용하는데, 이때 getter를 사용하기 때문에 없으면 에러가 발생한다는 걸 알게 되었다. 
    - 참고 글: https://qkrqkrrlrl.tistory.com/74#google_vignette
5. 직렬화 과정에 대해 간단히 학습했다.
    - ResponseEntity<객체>를 반환할 때, 클라이언트가 읽을 수 있는 형태로 변환한다.
    => 해당 객체의 public getter를 호출해 데이터를 읽어 JSON 문자열로 변환한다. 
```

### 3. 전략 평가

- 효과적이었던 것과 그 이유
    - 코드를 직접 작성하면서 발생한 문제를 해결하니, 학습하는 재미가 있었다.
- 비효과적이었던 것과 그 이유
    - 최대한 AI를 활용하지 않고 왜 테스트가 실패하는지 확인하고 싶었는데, 혼자 아무리 찾아봐도 모르겠어서 결국 AI에게 코드를 보내 확인했다.  
      스스로 문제 원인을 찾지 못한 점이 아쉬웠다.
- 막힌 것의 종류(1번)와 전략의 궁합은 어땠는가?
    - 테스트를 통과시키기 위한 정도로만 학습하니, 시간이 적게 걸리고 문제 해결 과정이 즐거워서 궁합이 좋았다.
