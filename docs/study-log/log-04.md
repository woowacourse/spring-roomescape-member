## 학습 로그 #4

**시간**: 04/29 19:00 ~ 22:10 (약 190분)
**학습 범위**: 4단계 계층 분리

### 단계 전환 체크
- 이전 단계에서 쓴 접근법을 그대로 쓸 것인가, 바꿔볼 것인가?
    - 휴식 시간을 통해 성찰하는 방식을 유지하되, 주기의 변화를 줄 것이다.
- 바꾼다면 왜? 바꾸지 않는다면 왜?
    - 정해진 주기마다 휴식하는 것은 쉬운 일이지만, 내 심리 상태를 점검하는 능력을 기르지 못한다.
    - 학습하며 지속적으로 내 심리 상태를 점검하고, 휴식이 필요한 시점에 휴식하도록 바꿔 볼 것이다.
- 이전 단계에서 발견한 "나에게 맞는 전략"이 새 단계에서도 통할 것 같은가?
    - 해야 할 작업이 다양하고 복잡할수록, 중간에 성찰하여 진행 상황을 점검하는 것은 효과적이라 예상한다.
    - 이번 단계에서는 계층 구조를 개편하고 리팩터링을 수행할 예정이니만큼, 잘 통할 것이다.

### 2. 이번 타임의 학습 전략
- 이전에 바꾸기로 한 전략은 무엇이었고, 실행했는가?
  - 심리 상태를 주기적으로 관찰하고, 휴식이 필요할 때 휴식한다.
  - 희망한 만큼 잘 실행하진 못했지만, 계속 의식하면서 내 심리 상태를 추적했다.
- 실제로 어떻게 학습했는지 디테일한 과정을 써보세요.
  - 혼자서 계속 구현하니, 한 번 학습해서 익숙해진 기술(JdbcTemplate 등)만 계속 사용했다.
  - 구현은 충분히 가능했지만 코드를 더 개선하고 싶었고, 다른 구현 방식을 살펴보는 것이 가장 빠른 학습 방법이라 판단했다.
  - 주변 크루들과 코드를 공유하며 어떤 기술을 어떻게 사용했는지 관찰했고, 내 방식과 비교해 이점이 큰 방식(SimpleInsertJdbc 등)들을 도입했다.
  - 이 과정까지 마친 후, 소크라테스 AI를 활용해 내 코드가 지닌 문제를 고민하고 개선함을 반복했다. 단, 완벽보단 완성을 추구하기 위해 30분의 시간 제한을 두었다.
  - 이 과정에서 내가 집중력을 유지하며 학습하고 있는지를 주기적으로 검토했고, 집중력이 떨어진다 느껴지는 시점에 5분의 휴식을 가지고 진행 상황을 정리했다.

### 3. 전략 평가
- 효과적이었던 것과 그 이유
  - 크루들과의 코드 공유
  - 기존 방식으로도 충분히 구현할 수 있었기에, 어떤 새로운 방식을 찾아야 할지 막막했다.
  - 나와 같은 문제를 다르게 해결한 크루들은 주변에서 쉽게 찾을 수 있었기에, 아주 빠르게 새로운 방식을 도입할 수 있었다.
- 비효과적이었던 것과 그 이유
  - 소크라테스 AI를 활용한 코드 개선
  - 긴 시간을 두고 내가 고민하여 코드를 개선하니, 마냥 수용하지 않고 비판적으로 생각하며 코드를 개선할 때는 효과적이었다.
  - 하지만, 우아한테크코스에서 배웠던 "완벽보다 완성을 추구하라"라는 철학과는 맞지 않았다.
  - AI를 활용해 고민하는 것도 좋지만, 이에 의존하는 순간 완벽한 구현에 욕심이 나, 적절히 타협하여 완성하기 많이 어려워졌다.
- 막힌 것의 종류(1번)와 전략의 궁합은 어땠는가?
  - 코드가 어딘가 마음에 들진 않지만, 어떻게 개선해야 하는지 알아내기 어려웠다.
  - AI나 구글링을 활용해서도 해결할 수 있었겠지만, 크루와 코드를 공유하는 것이 훨씬 빨리 해결할 수 있으리라 판단한다.
