### 2. 이번 타임의 학습 전략

- **이전에 바꾸기로 한 전략은 무엇이었고, 실행했는가?**
  지난번 로그에서 예고한 대로, 단순 구글링을 넘어 **'AI 도구를 학습 및 구현의 보조 도구(동료)로 활용하기'**를 처음으로 시도했다.
- **실제로 어떻게 학습했는지 디테일한 과정을 써보세요.**
  기존의 예약 시간 문자열을 별도의 테이블과 외래 키로 분리하면서 데이터베이스 정규화가 일어났다. 단순히 쿼리문만 작성하는 것이 아니라, 예약 객체가 시간 객체를 포함하는 구조를 유지하면서 데이터베이스 매핑을
  어떻게 깔끔하게 정의할지 고민이 컸다. 이를 해결하기 위해 AI에게 "내부 조인 결과를 복합 객체로 매핑하는 모던한 패턴"에 대해 질문했고, 제시받은 코드 구조를 바탕으로 식별자와 조인된 데이터를 조립하는 로직을
  구현했다. 또한, 기존 테스트가 깨지지 않도록 데이터를 먼저 세팅하고 식별자를 넘기는 흐름을 설계하는 데 AI의 조언을 참고했다.

### 3. 전략 평가

- **효과적이었던 것과 그 이유**
  **복합적인 문제 해결 시간 단축**. 구글링은 파편화된 정보를 제공하지만, AI는 내 현재 코드 구조를 이해한 상태에서 맞춤형 가이드를 주기 때문에 도메인 간 의존 관계를 처리하는 로직을 짜는 데 매우
  유용했다.
- **비효과적이었던 것과 그 이유**
  AI가 제안한 초기 코드 중 일부가 나의 유틸리티 클래스 스펙과 맞지 않아 컴파일 에러가 발생했다. 주도적인 AI 활용을 지향하는 만큼, AI의 답변을 맹신하기보다 내 프로젝트의 맥락에 맞게 '검증'하고 '
  수정'하는 과정이 반드시 수반되어야 함을 다시금 느꼈다.
- **막힌 것의 종류(1번)와 전략의 궁합은 어땠는가?**
  매우 훌륭했다. 특히 "이게 맞는 건지 모르겠다"는 설계적 고민에 대해 객체지향적인 대안을 제시받을 수 있어 심리적인 안정감과 기술적 성장을 동시에 얻을 수 있었다.

### 4. AI 피드백

- **자신의 학습 전략에 대해 AI 학습 전문가에게 피드백을 요청하고, 유용했던 제안 1가지 이상 기록**
    1. **컨텍스트 주도 질문 (Context-Driven Prompting)**: AI를 사용할 때 단순히 "코드 짜줘"가 아니라, "현재 내 테이블 구조는 A고 도메인 클래스는 B인데, 이 둘을 매핑할 때
       발생할 수 있는 잠재적 문제는 뭐야?"와 같이 제약 사항을 명확히 전달한 점이 매우 좋은 접근이었다.
    2. **점진적 추상화 (Incremental Abstraction)**: 지금은 컨트롤러에서 모든 로직을 처리하고 있지만, AI는 이 과정에서 '계층 분리'의 필요성을 미리 언급해 주었다. 구현에 급급해
       놓칠 수 있는 다음 단계의 핵심 키워드를 미리 학습 레이더에 넣어준 점이 유용했다.
