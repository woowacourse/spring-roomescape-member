### 1. 막힌 것의 종류
이번에 막힌 것은 어떤 종류의 어려움이었는가? (해당하는 것에 체크)
- [ ] 개념 자체를 모르겠다 (예: "스프링 빈이 뭔지 모르겠다")

- [ ] 코드는 돌아가는데 이게 맞는 건지 모르겠다 (예: "계층 분리를 이렇게 해도 되나?")
- [X] 기타: 새로운 방식 도입의 순서가 꼬여서 롤백해야 했다(time을 구현 완료하지 않은 채로, Reservation연동과 time 구현을 동시에 하려다 문제가 생겼다), 타고타고 문제가 생겨서(다른 할 일로 이동한 게 아니라, 버그가 발견되어 계속 연결되어 해결함) 깊게 들어가야 했는데 시작 지점으로 돌아올 때 좀 헤맴

### 4. AI 피드백
- **인지적 빵부스러기: "스택 메모 (The Breadcrumb Stack)"**
   > 버그를 해결하러 깊게 들어갈 때(Rabbit-holing), 원래 어디서 시작했는지 뇌가 잊지 않도록 기록하는 방법

    - 방법: 버그 해결을 위해 현재 하던 일을 멈추고 깊이 들어가야 할 때, 종이나 메모장 맨 위에 현재 위치와 목적지를 딱 한 줄 적는다.
    - 예: [Reservation 연동 중] -> [Time 생성자 에러 수정 중] -> [SQL 문법 확인 중]

- **공식 문서의 "경고문 포착" (Hunting for Warnings)**
    > 이제 공식 문서를 보는 법을 깨우치셨으니, 한 단계 더 나아가 보자
    - 방법: 공식 문서를 볼 때 '사용법(How-to)'만 보지 말고, '주의(Warning)', '중요(Important)', '참고(Note)' 박스만 먼저 찾아 읽으세요.
