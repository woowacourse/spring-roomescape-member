### 2. 이번 타임의 학습 전략

- 이전에 바꾸기로 한 전략은 무엇이었고, 실행했는가?
    1. 학습 시간을 꼭 정해놓지는 않고 기능 단위로 학습하기
    2. 코드로 먼저 접하고 개념 학습하기

- 실제로 어떻게 학습했는지 디테일한 과정을 써보세요.
    - KeyHolder 사용법, jdbcTamplate을 익히기 위해 AI에게 우리 미션 상황에 맞는 예시 코드를 먼저 요청해 구조를 파악했다.
    - LMS에서 제공하는 JdbcTemplate 학습 테스트를 수행한 후에 미션을 진행해서 훨씬 수월하게 미션을 진행할 수 있었다.
    - 코드를 작성하며 RowMapper의 타입 변환을 해야하는지, 해야하면 어떻게 하는 지 막히는 부분이 있었는데, 이럴 때 공식 문서를 선택적으로 참조했다.
      공식문서로도 해결이 되지 않는다면 너무 오래 혼자 생각하지 않고 AI에게 힌트나 필요한 자료를 달라고 했다.

### 3. 전략 평가

- 효과적이었던 것과 그 이유
    - 예시 코드 먼저 보고, 선택적으로 문서를 학습하는 역방향으로 학습하기가 효과적이었다.
      문법이 생소한 경우에도 어떻게 동작하는 지 쉽게 구조를 파악할 수 있었다. 그렇다보니 자연스럽게 문서를 읽었을 때에 개념이 더 쉽게 학습되었다.
    - 기능 단위로 학습을 끊어서 필요한 만큼 학습하는 데 도움이 되었다.
- 비효과적이었던 것과 그 이유
    - 딱히 없었다.

- 막힌 것의 종류(1번)와 전략의 궁합은 어땠는가?
- 생소한 문법을 코드로 옮길 때 역방향 학습과 궁합 좋았다. 예시 코드로 구조를 먼저 눈에 익히니 개념을 공부할 때, 코드를 작성할 때 막히는 느낌 없이 진행됐다.
  다만 구조 이해에는 효과적이지만, 동작 원리까지 깊게 이해하려면 추가 학습이 필요하다. 또한 다른 개념과 결합된 응용 상황에서는 다시 막힐 수 있다.

### 4. AI 피드백

“개념은 아는데 코드로 못 옮기는 문제”에 대해 예시 코드 → 개념 학습 전략이 잘 맞았고, 실제로 구현 과정의 막힘을 줄이는 데 효과적이었다.
특히 공식 문서와 AI를 상황에 맞게 선택적으로 활용한 점이 효율적인 학습 흐름으로 보인다.
전략의 한계도 잘 짚었다. 구조 이해에는 효과적이지만, 동작 원리까지 충분히 이해하지 않으면 이후 응용 상황에서 다시 막힐 수 있다.
다음 단계에서는 코드의 일부를 직접 설명해보거나, 다른 방식과 비교해보는 식으로 “이해 깊이”를 보완하는 것이 필요하다.

### 5. 다음 타임에 바꿀 것

- 유지할 것과 그 이유
  예시 코드 → 개념 순서로 학습하는 방식은 구현 흐름을 빠르게 파악하는 데 효과적이어서 유지한다.
  공식 문서와 AI를 상황에 맞게 선택적으로 활용하는 방식도 효율적이어서 유지한다.

- 바꿀 것과 그 이유
  응용 상황에서 다시 막히는 것을 방지하기 위해서 코드 구조만 이해하는 데서 그치지 않고, 일부 코드에 대해 “왜 이렇게 동작하는지” 설명해보는 과정을 추가한다.
  스스로 이해하는 힘을 키우기 위해 AI를 사용하기 전에 짧게라도 먼저 시도해보고, 이후에 활용하는 방식으로 보완한다.
