## 학습 이유
Spring Boot로 API를 개발하던 중, @RequestMapping과 @GetMapping을 하나의 메서드에 같이 썼다가 모호한 매핑 오류가 발생했다. 

## 정보 수집
- 공식 문서: @RequestMapping - https://docs.spring.io/spring-framework/docs/current/javadoc-api/org/springframework/web/bind/annotation/RequestMapping.html
	- 유연한 메서드 시그니처를 가진 요청 처리 클래스의 메서드에 웹 요청을 매핑하기 위한 어노테이션
	- 클래스 수준과 메서드 수준 모두에서 사용할 수 있음
	- 이 어노테이션은 동일한 요소(클래스, 메서드 등)에 선언된 다른 @RequestMapping 어노테이션과 같이 사용할 수 없음. @GetMapping, @PostMapping과 같은 합성 어노테이션과도 불가

- 공식 문서: @GetMapping - https://gemini.google.com/app/25de3e29b9a36cde?utm_source=app_launcher&utm_medium=owned&utm_campaign=base_all
	- HTTP GET 요청을 특정 핸들러 메서드에 매핑하기 위한 어노테이션
	- RequestMapping(method = RequestMethod.GET)의 축약형으로 동작하는 합성 어노테이션

- 합성 어노테이션이란?
	- 하나 이상의 기존 어노테이션을 조합해 만든 새로운 어노테이션
	- @GetMapping의 메타 어노테이션에 @RequestMapping(method = RequestMethod.GET)이 포함되어 있음

- RequestMapping과 GetMapping을 같이 쓰면 설정이 병합되거나 중복 정의로 처리됨
	- 클래스 단위(RequestMapping) + 메서드 단위(GetMapping) 모두 정의가 될 경우 계층적으로 합쳐져 경로를 생성
	- 하나의 메서드의 두 개를 동시에 쓰는 경우 모호한 매핑 오류(Ambiguous mapping)가 발생

## 나만의 언어 (구체적 예시 포함)
@GetMapping은 @RequestMapping을 메타 어노테이션으로 가지고 있다. 그래서 하나의 메서드에 @RequestMapping과 @GetMapping을 같이 사용할 경우, 매핑 정보가 중복되어 모호한 매핑 오류가 발생한다. 

모호한 매핑 오류는 상사 2명이 부하 직원 1명에게 동시에 명령을 할 때, 누구의 말을 들어야 할지 몰라서 헷갈리는 경우라고 볼 수 있다. 

## 나만의 언어 검증
- (오개념) 모호한 매핑 오류는 두 어노테이션을 한 메서드에 동시에 쓸 때 반드시 발생하는 게 아니라, 서로 다른 두 개의 메서드가 같은 경로를 처리하려고 할 때 발생함
	- 나는 지금까지 @RequestMapping과 @GetMapping을 같이 쓴 게 문제라고 생각했는데, GET API와 POST API 모두 @RequestMapping을 같은 경로로 설정했기에 발생한 오류였다!
	
## 결론
모호한 매핑 오류는 서로 다른 메서드가 같은 경로를 처리하려고 할 때 발생한다. 
HTTP 메서드가 다르면 발생하지 않는다. 

예를 들어 @RequestMapping("/home")을 create 메서드에도 붙이고 read 메서드에도 붙이면, "/home" 경로로 요청이 들어왔을 때 어떤 메서드를 실행해야 할지 혼선이 오기 때문에 오류가 발생하는 것이다.
