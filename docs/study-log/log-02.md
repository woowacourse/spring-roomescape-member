```
# 실제 학습 과정
- JdbcTemplate을 사용하여 insert, select, delete를 구현했다.
  insert 시 AUTO_INCREMENT id를 가져오기 위해 KeyHolder를 사용해야 한다는 것을 알게 되었다.
  ResultSet에서 LocalDate, LocalTime으로 매핑하는 과정에서 getObject를 사용해야 한다는 것을 학습했다.
- DB 연동 후 테스트를 진행했는데, 테스트 간 데이터가 누적되는 문제가 발생했다.
  처음에는 @DirtiesContext로 해결하려 했지만 DB 자체는 초기화되지 않는다는 것을 알게 되었다.
  file 기반 H2를 사용하고 있었기 때문에 테스트마다 데이터가 유지되는 구조라는 것을 이해했다.
  테스트용 mem DB를 따로 설정하여 환경을 분리하는 방식으로 해결했다.
- schema.sql이 실행되지 않는 문제를 겪었고, spring.sql.init.mode=always 설정이 필요하다는 것을 알게 되었다.

# 후기
- JPA를 사용해본 경험이 있기때문에 repository의 역할을 이해하는 것이 크게 어렵진 않았다.
  하지만 JdbcTemplate는 처음이라 구문을 이해하기 위해서 서칭, ai를 사용했다.
  ResultSet이나 KeyHolder등은 정확히 몰랐지만 Public api까지만 이해하면서 활용은 할 수 있었다.
  물론 내부적으로 어떤 일이 일어나는지 궁금했지만 이번 단계에서는 그렇게하지 않았다.
- 테스트와 운영 DB는 분리되어야한다는 사실을 알고있었지만, test/resources/application.properties만 설정하면 자동으로 되는지는 몰랐다.
  일단 문제를 해결하기위해 서칭한 결과대로 mem h2를 사용한다는 test/resources/application.properties설정을 만들었고 해결했다.
  요구사항 구현이 다 끝나고 어떻게 바로 test db 세팅을 불러올 수 있는지 알아봤다.
- Spring Boot에서 db table을 생성하지 못하는 문제가 있었다.
  schema.sql이 실행되고있지 않음을 깨닫고 문제를 해결하려고 했다.
  mem h2가 아닌 file h2는 table init을 명시해줘야한다고 한다. (spring.sql.init.mode=always)
  우선 요구사항을 만족해야하니 이렇게만 알고 넘어갔다.
- 전체적으로, 쭉 진행하기위해서 deep한 내용은 보지않았고 활용이 가능한가에 집중했다.
```
