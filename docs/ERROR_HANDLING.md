# 오류 처리 가이드 (Error Handling Guide)

이 문서는 방탈출 예약 시스템 API에서 발생하는 오류 응답의 구조와 정책을 설명합니다.

## 1. 개요

본 프로젝트는 모든 에러 응답에 대해 일관된 JSON 구조를 사용합니다. 응답의 `Content-Type`은 `application/json`입니다.

## 2. 에러 응답 구조

모든 에러 응답은 아래와 같은 공통 필드를 포함합니다.

| 필드명 | 타입 | 설명 |
| :--- | :--- | :--- |
| **errorType** | `String` | 에러의 성격 (예: `BUSINESS`, `SERVER`) |
| **message** | `String` | 사용자에게 보여줄 에러 메시지 |

### 응답 예시
```json
{
  "errorType": "BUSINESS",
  "message": "해당 번호의 예약 정보를 찾을 수 없습니다."
}
```

---

## 3. HTTP 상태 코드별 응답

| 상태 코드 | 의미 | 설명 |
| :--- | :--- | :--- |
| **200 OK** | 성공 | 요청이 정상적으로 처리된 경우입니다. |
| **400 Bad Request** | 잘못된 요청 | 필수 파라미터 누락, 형식 오류 등 요청 자체가 잘못된 경우입니다. |
| **403 Forbidden** | 권한 없음 | 관리자 전용 API에 유효한 권한(Header) 없이 접근한 경우입니다. |
| **404 Not Found** | 리소스 없음 | 요청한 데이터가 존재하지 않거나, 잘못된 API 경로로 요청한 경우입니다. |
| **409 Conflict** | 비즈니스 규칙 위반 | 중복 예약, 사용 중인 테마/시간 삭제 시도 등 데이터 충돌 발생 시 사용합니다. |
| **422 Unprocessable Entity** | 유효하지 않은 데이터 | 과거 날짜로 예약 시도 등 데이터의 논리적 유효성이 떨어지는 경우입니다. |
| **500 Internal Server Error** | 서버 오류 | 서버 내부에서 예상치 못한 에러가 발생한 경우입니다. |

---

## 4. 주요 에러 메시지

### 4.0. 요청 공통 (400, 403, 405)
- `요청 정보가 올바르지 않습니다. 입력 내용을 다시 확인해주세요.` (400)
- `요청을 수행하기 위해 필요한 권한이 없습니다` (403)
- `지원하지 않는 요청 방법입니다. 주소 혹은 요청 방식을 확인해주세요.` (405)

### 4.1. 예약 관련 (Reservation)
- `해당 번호의 예약 정보를 찾을 수 없습니다.` (404)
- `입력하신 성함의 예약 정보를 찾을 수 없습니다.` (404)
- `예약자 성함이 누락되었습니다. 이름을 입력해주세요.` (400)
- `예약 날짜가 선택되지 않았습니다. 날짜를 선택해주세요.` (400)
- `예약 시간이 선택되지 않았습니다. 시간을 선택해주세요.` (400)
- `변경할 예약 날짜가 누락되었습니다. 날짜를 선택해주세요.` (400)
- `변경할 예약 시간이 누락되었습니다. 시간을 선택해주세요.` (400)
- `동일한 날짜, 시간, 테마의 예약이 이미 존재합니다. 다른 조건을 선택해주세요.` (409)
- `지나간 날짜나 시간으로는 예약을 생성하거나 삭제할 수 없습니다. 현재 이후의 시간을 선택해주세요.` (422)

### 4.2. 시간 관련 (Time)
- `선택하신 예약 시간 정보가 존재하지 않습니다.` (404)
- `사용 중인 예약 시간은 삭제할 수 없습니다. 관련 예약을 먼저 삭제해주세요.` (409)

### 4.3. 테마 관련 (Theme)
- `선택하신 테마 정보가 존재하지 않습니다.` (404)
- `테마 이름이 비어있습니다. 이름을 입력해주세요.` (400)
- `테마 설명이 비어있습니다. 설명을 입력해주세요.` (400)
- `테마 썸네일 주소가 비어있습니다. 주소를 입력해주세요.` (400)
- `사용 중인 테마는 삭제할 수 없습니다. 관련 예약을 먼저 삭제해주세요.` (409)
