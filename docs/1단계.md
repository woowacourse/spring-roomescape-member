# API

## 테마 리스트 조회
1. 관리자는 등록된 테마들을 조회한다.

```http request
Method: GET
Endpoint: /themes
Content-Type: application/json
Request Body: 없음

Response: 200 OK
Content-Type: application/json
Response Body:
[
 {
    "id": 1,
    "name": "브라운",
    "description": "방탈출 1",
    "thumbnailUrl": "http://www.google.com/something1.png"
 },
 {
     "id": 1,
    "name": "초록",
    "description": "방탈출 2",
    "thumbnailUrl": "http://www.google.com/something2.png"
 }
]
```

## 테마 추가

1. 관리자는 이름, 설명, 이미지 URL을 작성한다.
2. 관리자는 테마를 추가한다.

```http request
Method: POST
Endpoint: /themes
Content-Type: application/json
Request Body: {
    "name": "브라운",
    "description": "설명",
    thumbnailUrl: "http://www.google.com"
}

Response: 201 CREATED
Content-Type: application/json
Response Body: {
    "id": 1,
    "name": "브라운",
    "description": "설명",
    thumbnailUrl: "http://www.google.com"
}
```

## 테마 삭제

1. 관리자는 테마 리스트를 확인한다.
2. 관리자는 테마를 선택한다.
3. 관리자는 선택된 테마를 제거한다.

```http request
Method: DELETE
Endpoint: /themes/{themeId}
Content-Type: application/json
Request Body: 없음

Response: 204 NO CONTENT
ResponseBody: 없음
```

# 고민해본 점

#### 요청을 보낼 때

Theme 관련 요청 보내는 방법

1. id에 대해서 AUTO_INCREMENT
2. id를 body와 함께
3. 클라이언트는 id를 보내지 않고 서버에서 UUID로 id를 추가

#### POST의 응답

- AUTO_INCREMENT를 하고, POST 요청인 경우에는 Response body에 테마 정보를 넣어서 클라이언트로 반환한다.
- Response Body를 꼭 보내야 하는가? 아니라면 언제 보내야 하는가?

-> 선택: AUTOINCREMENT로 하고, Response body에는 정보를 넣어서 반환한다.

#### 삭제하는 경우 Http Status Code
- 삭제를 하는 경우에 데이터가 존재하는 경우에는 204 NO CONTENT를 하면 되는데, 존재하지 않는 데이터를 삭제하는 경우에는 204 NO CONTENT인가 아니면 404 NOT FOUND인가?
- 선택: 204 NO CONTENT를 사용한다
- 이유: 존재하는 데이터를 삭제하든, 존재하지 않는 데이터를 삭제하든 결국에는 존재하지 않는 데이터이며, 통일성을 확보하기 위해서.
