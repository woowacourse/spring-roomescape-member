# 2단계 방탈출 사용자 예약

1. 사용자는 테마 목록을 조회한다.
2. 사용자는 테마를 선택한다.
2. 사용자는 날짜를 선택한다. (날짜랑 시간은 한 화면에서 표시)
3. 사용자는 선택한 테마와 날짜를 기반으로 예약 가능한 시간을 조회한다.
4. 사용자는 예약 가능한 시간을 선택한다.
5. 사용자는 예약한다.

## 과거 날짜 검증은 누가하는가?

1. 도메인 Reservation이 검증한다.
2. Service 자체에서 검증한다.

**결정**: Service 자체에서 검증한다.

**이유**: 정책상 사용자는 과거 날짜에 예약할 수 없고, 관리자는 과거 예약 복구나 정정을 위해 과거 날짜에 예약을 추가할 수 있도록 해야 한다고 판단했다. 이렇게 되면 도메인이 과거 날짜 판정을 하면 덩달아
사용자와 관리자 여부까지 알아야 한다.

그렇기에 비즈니스 목적별로 과거 시간 판정 여부를 분리하기 위해 service에서 검증하기로 했다.
