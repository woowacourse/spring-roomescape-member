# 3. 화면 목록 정의

UX 편의성을 고려하여 화면을 최소화하고, 한 화면에서 유기적으로 조작할 수 있도록 설계했습니다.

## 1. 사용자 화면 (Client)

### 1.1. 사용자 메인 화면 (예약하기)
- **경로**: `/`
- **목적**: 테마와 날짜를 한 화면에서 선택하여 예약 가능한 시간을 확인하고 곧바로 예약을 생성한다.
- **특징**: 달력 위젯과 테마 목록 위젯이 나란히 배치되며, 두 조건이 모두 선택되면 하단에 시간 슬롯 버튼들이 동적으로 나타남.

### 1.2. 내 예약 관리 화면
- **경로**: `/my-reservations`
- **목적**: 본인의 이름으로 조회한 예약 내역을 확인하고, 예약 취소 및 일정 변경을 수행한다.
- **특징**: 
  - 화면 진입 시 이름 입력 폼 노출.
  - 이름 검색 시 하단에 예약 카드 리스트 형태로 내역 표시.
  - 각 카드에 [일정 변경], [예약 취소] 버튼 제공.
  - [일정 변경] 클릭 시 모달(Modal) 창이 떠서 화면 이동 없이 날짜/시간을 수정할 수 있도록 처리.

## 2. 관리자 화면 (Admin)

관리자 화면은 공통 사이드바(LNB)를 가진 하나의 레이아웃 템플릿 안에서 컨텐츠만 교체되는 SPA 형태로 동작합니다.

### 2.1. 관리자 예약 관리 화면 (기본 관리자 홈)
- **경로**: `/admin` (또는 `/admin/reservations`)
- **목적**: 전체 사용자들의 예약 내역을 페이징하여 리스트 형태로 조회하고 악성 예약을 삭제(취소)한다.

### 2.2. 관리자 테마 관리 화면
- **경로**: `/admin/themes`
- **목적**: 새로운 방탈출 테마를 추가하거나 기존 테마를 삭제한다.

### 2.3. 관리자 시간 관리 화면
- **경로**: `/admin/times`
- **목적**: 예약 가능한 기준 시간(슬롯)을 추가하거나 삭제한다.

## 3. 화면 이동 흐름 (Flow)

- `[사용자 메인]` --(예약 완료)--> `[내 예약 관리]` (예약 확인 유도)
- `[내 예약 관리]` --(일정 변경 클릭)--> `[일정 변경 모달]` --(완료)--> `[내 예약 관리 새로고침]`
- 화면 최상단 네비게이션(GNB)의 `[관리자 모드]` 버튼 클릭 --> `[관리자 예약 관리]` 화면 진입
- 관리자 화면 내 좌측 메뉴(LNB) 클릭 --> 각 관리자 메뉴 간 즉시 이동
