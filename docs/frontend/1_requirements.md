# 1. 요구사항 분석

## 1. 액터 목록
- **사용자**: 방탈출 예약을 조회, 생성, 변경, 취소하는 일반 고객
- **관리자**: 방탈출 테마, 예약 시간, 전체 예약 내역을 전반적으로 관리하는 운영자

## 2. 기능 목록

### 사용자 (Client)
- **테마 조회**: 예약 가능한 전체 테마 목록을 확인한다.
- **인기 테마 조회**: 인기 있는 테마 순위를 확인한다. (기본 7일 기준, Top 10)
- **예약 가능 시간 조회**: 선택한 날짜와 테마에 대해 예약 가능한 시간대 목록을 확인한다.
- **예약 생성**: 원하는 날짜, 테마, 시간을 선택하고 이름 등의 정보를 입력하여 예약을 생성한다.
- **내 예약 조회**: 본인의 이름으로 등록된 예약 내역을 조회한다.
- **예약 변경**: 기존 예약의 일정을 변경한다.
- **예약 취소**: 등록된 예약을 취소한다.

### 관리자 (Admin)
- **테마 관리**: 새로운 테마를 등록하거나 기존 테마를 삭제한다. (이름, 설명, 썸네일 이미지 등)
- **예약 시간 관리**: 예약 시간(슬롯)을 추가하거나 삭제한다.
- **전체 예약 관리**: 전체 예약 내역을 페이징 단위로 조회하고, 필요한 경우 관리자 권한으로 예약을 취소(삭제)한다.

## 3. 핵심 정책/제약 조건
- **예약 중복 불가**: 동일한 날짜(Date), 예약 시간(Time), 테마(Theme)에는 단 하나의 예약만 가능하다.
- **예약 정보 식별**: 현재는 로그인 기반이 아니며, 사용자의 '이름'을 기준으로 예약을 조회하고 관리한다.
- **시간/테마 종속성**: 예약은 반드시 서버에 등록된 '테마'와 '예약 시간(Time 슬롯)'을 기준으로 이루어진다.
- **예약 취소 정책**: 사용자는 `cancellation` 상태로 변경/취소하고, 관리자는 예약을 아예 삭제(`DELETE`)하는 방식으로 동작한다.
