# 5. API 사용 흐름 정의

모든 API 호출 시 UX를 고려하여 **연타 방지(disabled)**와 **커스텀 피드백 UI(토스트 알림 또는 에러 텍스트)**를 반영한 흐름입니다.

## 1. 예약하기 (사용자 메인 화면)

### 1.1. 초기 테마 목록 조회
- **트리거**: 메인 페이지 초기 진입 시
- **API 호출**: `GET /themes` (전체 테마), `GET /themes/rank` (인기 테마)
- **성공 시 화면**: 가져온 테마 정보로 화면의 테마 리스트 컴포넌트 렌더링
- **실패 시 화면**: "테마 목록을 불러오지 못했습니다"라는 안내 텍스트 노출

### 1.2. 예약 가능 시간 조회
- **트리거**: 날짜(달력)와 테마를 모두 선택 완료했을 때
- **로딩 처리**: 시간 슬롯 영역에 로딩 스피너 표시
- **API 호출**: `GET /reservation-times/available?themeId={themeId}&date={date}`
- **성공 시 화면**:
  - 남은 시간 버튼 렌더링
  - 이미 예약된 시간은 시각적으로 비활성화(`disabled` 및 회색 처리)하여 UX 개선
- **실패 시 화면**: "예약 가능한 시간을 확인할 수 없습니다" 텍스트 노출

### 1.3. 예약 확정 (생성)
- **트리거**: 이름 입력 후 최종 '예약하기' 버튼 클릭
- **로딩 처리**: '예약하기' 버튼 비활성화(`disabled`) 및 텍스트 '예약 중...'으로 변경 (연타 방지)
- **API 호출**: `POST /reservations` (Body: date, timeId, themeId, name)
- **성공 시 화면**: 
  - 화면 하단에 "예약이 성공적으로 완료되었습니다" **토스트(Toast) 알림** 표시
  - 1~2초 후 내 예약 조회 페이지(`/my-reservations`)로 자동 이동
- **실패 시 화면**: 
  - 토스트 알림으로 에러 메세지(예: "누군가 먼저 예약한 시간입니다") 노출
  - 버튼 활성화 복구하여 재시도 가능하도록 처리

## 2. 내 예약 조회 및 관리

### 2.1. 예약 내역 검색
- **트리거**: 이름 입력 폼에서 '조회' 버튼 클릭
- **로딩 처리**: 조회 버튼 비활성화
- **API 호출**: `GET /reservations?name={name}`
- **성공 시 화면**: 
  - 예약된 내역 리스트 렌더링
  - 내역이 없을 경우 빈 상태(Empty State) UI 표시 ("예약된 내역이 없습니다")
- **실패 시 화면**: 입력 폼 하단에 붉은색 텍스트로 "조회 중 문제가 발생했습니다" 노출

### 2.2. 일정 변경
- **트리거**: 변경 모달창에서 새 일정 선택 후 '변경하기' 클릭
- **로딩 처리**: 변경 버튼 비활성화
- **API 호출**: `PUT /reservations/{id}/schedule` (Body: date, timeId, themeId)
- **성공 시 화면**: 모달 창 닫음 + 토스트 알림("일정이 변경되었습니다") + 화면 리스트 재조회하여 갱신
- **실패 시 화면**: 모달 창 내부에 붉은색 에러 텍스트 노출

### 2.3. 예약 취소
- **트리거**: 개별 예약 내역의 '취소' 버튼 클릭
- **로딩 처리**: 해당 취소 버튼 비활성화
- **API 호출**: `PUT /reservations/{id}/cancellation`
- **성공 시 화면**: 토스트 알림("예약이 취소되었습니다") 표시 후, 화면 리스트에서 해당 내역 DOM 제거
- **실패 시 화면**: 토스트 알림으로 취소 실패 안내
