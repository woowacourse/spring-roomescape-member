# 8. UI 상태 테스트 시나리오

## 1. 사용자 시나리오 (User Flow)
1. **정상 예약 흐름**: 메인 진입 ➡️ 테마 선택 ➡️ 날짜 선택 ➡️ 남은 시간 선택 ➡️ 이름 입력 ➡️ 예약 완료 후 토스트 노출 ➡️ 내 예약 목록으로 이동
2. **예약 변경 흐름**: 내 예약 조회 ➡️ 조회 버튼 클릭 ➡️ 목록 확인 ➡️ 특정 내역 변경 클릭 ➡️ 모달에서 새로운 시간 선택 ➡️ 변경 완료 확인
3. **예약 취소 흐름**: 내 예약 조회 ➡️ 취소 버튼 클릭 ➡️ 알림 노출 및 DOM에서 즉시 제거되는지 확인

## 2. 상태별 테스트 (UI 렌더링 검증)
1. **Loading 상태**: 데이터를 불러오는(fetch) 동안 화면에 스켈레톤이나 로딩 스피너가 표시되고, 버튼이 `disabled` 되어 연속 클릭을 막는가?
2. **Empty 상태**:
   - 테마와 날짜를 선택했는데 예약 가능한 시간이 하나도 없을 때 "예약 가능한 시간이 없습니다" 문구가 뜨는가?
   - 내 예약을 검색했을 때 예약 내역이 없으면 빈 상태(Empty State) UI가 노출되는가?
3. **Error 상태**: 
   - 폼 입력값이 잘못되었을 때(예: 빈 이름) 검증 에러 문구가 뜨는가?
   - 네트워크 오류 발생 시 에러 텍스트나 토스트가 노출되는가?
4. **Success 상태**: 예약/취소/수정 완료 시 팝업(Toast) 알림이 정상적으로 뜨는가?

## 3. API 성공/실패 케이스 검증
1. **예약 생성 실패**: 내가 예약 버튼을 누르기 직전에 다른 사람이 먼저 예약을 성공해서 400/409 에러가 뜰 경우, 에러 토스트가 뜨고 페이지가 멈추지 않는가?
2. **존재하지 않는 이름 조회**: 등록되지 않은 이름 검색 시 404 혹은 빈 리스트 처리가 크래시 없이 잘 동작하는가?
3. **취소 실패**: 이미 취소된 예약을 다른 창에서 동시에 취소하려 할 때, 에러 알림이 정상적으로 뜨는가?
