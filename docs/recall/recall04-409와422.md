# 409와 422의 차이

409 Conflict는 '대상 리소스 현재 상태'와 요청이 충돌한다는 뜻이다. 422 Unprocessable Content는 보낸 body가 문법이나 데이터 형식에는 문제가 없지만 의미상의 문제로 처리할 수 
없음을 뜻한다.

## 중복 예약을 등록할 수 없을 때는

사용자가 예약이 데이터베이스에 등록되기를 기대할 것이다. 하지만 예약이 동일 날짜, 시간, 테마에 등록될 수 없다는 조건과 충돌하여 예약 리소스가 가져야 하는 상태와 어긋나게 된다. 이 경우는 예약 컬렉션 
자체의 제약과 요청이 맞지 않아 거절한 것이므로 409로 본다.

## 클라이언트가 이상한 값을 주는 경우는 400인가 422인가?

400은 클라이언트 쪽 오류로 처리할 수 없다는 뜻이다. 422 역시 클라이언트가 보낸 데이터가 의미상 문제가 있어 처리할 수 없다는 의미니 422이 다루는 정보가 400이 다루는 정보의 부분집합이다.

이 경우는 400, 422 둘다 문제가 없다.
