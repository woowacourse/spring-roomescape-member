# Resource & Representation

## Resource

리소스는 http 요청의 대상이다. get이면 결과로 데이터를 얻기를 기대하고 post면 보낸 데이터가 등록되기를 기대할 것이다. 리소스는 이렇듯이 어떤 구체적 형태가 아닌 개념적으로 존재하는 것이다. 

리소스의 상태라는 것은 리소스 개별 상태 또는 리소스의 컬렉션이 어떻게 달라지느냐가 결정한다.

## Representation

representation은 이 리소스를 어떻게 표현할지를 나타낸다. REST API면 JSON 웹페이지면 html, 이미지면 jpg, png로 나타낼 수 있다.
