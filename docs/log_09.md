### 학습 과정

#### 의문의 시작

> "커넥션이 쓰레드에 바인딩된다는데, 어떻게 그게 가능한거야?"

처음엔 "메서드를 실행하면 흐름이 끝날 때까지 같은 쓰레드에서 동작하지 않을 수도 있겠다"고 생각했다.
→ 그런데 이 전제 자체가 잘못됐다. 아래에서 정리.

---

#### ThreadLocal

자바에서 쓰레드마다 독립적인 저장 공간을 제공하는 클래스.
쓰레드 객체 자체가 `ThreadLocalMap`을 들고 있어서, 같은 쓰레드면 항상 같은 Map에 접근한다.

```java
// ThreadLocal.get() 내부 (단순화)
public T get() {
    Thread t = Thread.currentThread();  // 현재 쓰레드 식별
    ThreadLocalMap map = t.threadLocals; // 쓰레드 객체가 Map을 들고 있음
    return map.get(this);
}
```

#### TransactionSynchronizationManager

Spring이 `ThreadLocal`을 이용해 커넥션을 관리하는 클래스.

```
@Transactional 시작
  → 커넥션 풀에서 커넥션 획득
  → TransactionSynchronizationManager(ThreadLocal)에 저장
  → Service, Dao 모두 같은 ThreadLocal에서 꺼내 사용
  → 트랜잭션 종료 시 커넥션 반납 + ThreadLocal 정리
```

#### 핵심 발견 — 질문의 전제가 잘못됐다

메서드 호출은 기본적으로 호출한 쓰레드에서 그대로 실행된다.

```
요청 → 쓰레드 1
  └── Controller  // 쓰레드 1
        └── Service  // 쓰레드 1
              └── Dao  // 쓰레드 1
```

새로운 쓰레드가 생기는 건 `@Async`, `new Thread()`, `ExecutorService` 같은 명시적 개입이 있을 때뿐이다.

→ **메서드를 실행하면 다른 함수를 호출하더라도 같은 쓰레드에서 동작하고, ThreadLocal에 커넥션이 바인딩되어 있기 때문에 커넥션 전파가 가능하다.**

#### 컨텍스트 스위치와의 관계

컨텍스트 스위치가 일어나도 쓰레드 객체는 힙 메모리에 그대로 존재한다.
CPU에서 잠깐 내려오는 것뿐이고, 쓰레드 객체와 `ThreadLocalMap`은 유지된다.
→ 컨텍스트 스위치와 상관없이 같은 쓰레드면 항상 같은 커넥션을 사용할 수 있다.

#### @Async와의 연결

이전에 배운 "@Async는 트랜잭션이 전파 안 된다"가 이제 근거를 갖게 됐다.
`@Async`는 다른 쓰레드를 사용 → 다른 `ThreadLocal` → 다른 커넥션 → 트랜잭션 전파 불가

#### 셀프 인보케이션과의 구분

셀프 인보케이션은 쓰레드 문제가 아니라 프록시 문제다.
쓰레드는 같으니까 `ThreadLocal` 커넥션은 공유되지만, 프록시를 거치지 않아서 `@Transactional` 설정이 무시된다.

---

#### 오늘의 연결고리

```
Thread.currentThread() → 쓰레드 식별
ThreadLocal → 쓰레드 객체에 붙은 독립 저장 공간 (힙에 존재)
컨텍스트 스위치 → 쓰레드 객체는 힙에 유지 → ThreadLocal 안전
TransactionSynchronizationManager → ThreadLocal로 커넥션 관리
→ 같은 쓰레드 = 같은 커넥션 = 같은 트랜잭션
```

## 3. 전략 평가

- 효과적이었던 것과 그 이유
  질문 자체의 전제가 잘못됐다는 걸 탐구 과정에서 스스로 발견했다.
  "메서드 호출은 같은 쓰레드에서 동작한다"는 당연한 사실을 명확히 인식하게 됐다.

- 비효과적이었던 것과 그 이유
  컨텍스트 스위치와 ThreadLocal의 관계를 처음엔 직관적으로 이해하지 못했다.
  "쓰레드 객체가 힙에 존재한다"는 연결이 부족했다.

- 막힌 것의 종류(1번)와 전략의 궁합은 어땠는가?
  잘못된 전제에서 출발한 질문이었는데, 추측 → 연결 질문 방식이 전제를 스스로 수정하게 도와줬다.

## 4. AI 피드백

오늘 가장 중요한 순간은 "질문 자체가 잘못됐다"는 걸 스스로 발견한 것이다.
잘못된 전제를 품은 질문은 아무리 답을 찾아도 해소되지 않는다.
탐구 과정에서 전제 자체를 의심하는 능력이 생겼다는 게 이번 사이클의 핵심 성장이다.
